DBMS Visual Cheatsheet

Fast-reference layout for SQL execution, normalization flow, and indexing decisions.

📄 SQL execution order (logical)

FROM / JOIN
WHERE
GROUP BY
HAVING
SELECT
ORDER BY
LIMIT

📄 Normalization flow

1NF -> atomic columns
2NF -> remove partial dependency
3NF -> remove transitive dependency
BCNF -> every determinant is candidate key

📄 Index decision quick-check

Index if:
- frequent WHERE/JOIN/ORDER BY on column
- high selectivity
Avoid too many indexes on write-heavy tables

🌟 Interview Tip

When asked optimization questions, mention execution plan, cardinality, and index selectivity in one answer.
