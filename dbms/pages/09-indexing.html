Indexing

Indexes, clustered vs non-clustered, and query performance impact.

📚 Concept

Index is a data structure (often B-Tree) that speeds up lookups by avoiding full table scans.

Clustered vs Non-clustered

  • Clustered index: data rows stored in index order (usually one per table).
  • Non-clustered index: separate structure pointing to rows (many allowed).

▶ SQL examples

CREATE INDEX idx_emp_dept ON employees(dept_id);
CREATE INDEX idx_emp_salary ON employees(salary);

-- Query benefiting from index
SELECT * FROM employees WHERE dept_id = 20;

⚖ Trade-off

  • ✅ Faster read queries
  • ❌ Slower writes (INSERT/UPDATE/DELETE) due to index maintenance
  • ❌ More storage usage

🌟 Interview Tips

  • Composite indexes follow left-prefix rule.
  • Indexing low-selectivity columns may not help much.
