Revision Chunk A (01-05)

Introduction, relational concepts, SQL overview, queries, and joins.

📚 10-Minute Recall

  • DBMS beats file systems on consistency, concurrency, security, and recovery.
  • Keys + constraints enforce relational correctness.
  • SQL categories: DDL/DML/DQL/DCL/TCL.
  • WHERE filters rows before grouping; HAVING filters groups.
  • Join type decides unmatched row behavior.

▶ Rapid Drill

  1. Difference between PK and candidate key?
  2. DELETE vs TRUNCATE?
  3. WHERE vs HAVING with one example each.
  4. LEFT JOIN result when right row is missing?
  5. How to return unique departments?

🌟 Interview Tip

When solving SQL questions, verbalize data flow: FROM/JOIN → WHERE → GROUP BY → HAVING → SELECT → ORDER BY → LIMIT.

Database engineering decisions should connect schema design, query behavior, indexing, transactions, and scalability constraints.
