Query Optimization

How databases execute queries efficiently and how to write performance-aware SQL.

📚 Optimizer basics

  • Parses SQL and generates execution plan.
  • Chooses join order, join algorithm, indexes, and access paths.
  • Uses statistics/cardinality estimates.

▶ Practical tuning checklist

1) Use EXPLAIN / EXPLAIN ANALYZE
2) Avoid SELECT * in large joins
3) Add indexes for join/filter columns
4) Filter early (push predicates)
5) Avoid functions on indexed columns in WHERE
6) Use appropriate data types
7) Batch writes and avoid chatty transactions

▶ Example

-- Less optimal
SELECT * FROM employees WHERE YEAR(join_date) = 2024;

-- Better (index-friendly range)
SELECT emp_id, emp_name
FROM employees
WHERE join_date >= '2024-01-01' AND join_date < '2025-01-01';

🌟 Interview Tips

  • Say “measure before optimize” and mention execution plan.
  • Discuss read/write trade-offs with indexing.

▶ Theory Deep Dive

Database engineering decisions should connect schema design, query behavior, indexing, transactions, and scalability constraints.
