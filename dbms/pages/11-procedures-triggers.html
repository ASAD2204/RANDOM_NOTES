Stored Procedures and Triggers

Server-side reusable logic and automatic event-driven actions.

📚 Stored Procedures

Precompiled SQL blocks that can accept parameters and encapsulate business logic.

▶ Procedure example (MySQL-style)

DELIMITER //
CREATE PROCEDURE GetEmployeesByDept(IN p_dept INT)
BEGIN
  SELECT emp_id, emp_name, salary
  FROM employees
  WHERE dept_id = p_dept;
END //
DELIMITER ;

CALL GetEmployeesByDept(10);

📚 Triggers

Triggers run automatically before/after INSERT, UPDATE, DELETE events.

▶ Trigger example

CREATE TABLE salary_audit(
  emp_id INT,
  old_salary DECIMAL(10,2),
  new_salary DECIMAL(10,2),
  changed_at TIMESTAMP DEFAULT CURRENT_TIMESTAMP
);

DELIMITER //
CREATE TRIGGER trg_salary_update
AFTER UPDATE ON employees
FOR EACH ROW
BEGIN
  IF OLD.salary <> NEW.salary THEN
    INSERT INTO salary_audit(emp_id, old_salary, new_salary)
    VALUES (NEW.emp_id, OLD.salary, NEW.salary);
  END IF;
END //
DELIMITER ;

🌟 Interview Tips

  • Keep triggers lightweight; avoid hidden side effects.
  • Prefer application/service logic for complex workflows unless strict DB-level enforcement is needed.
