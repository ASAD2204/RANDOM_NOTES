Views

Virtual tables based on SQL queries for abstraction and security.

📚 Concept

A view stores a query definition, not the data itself (except materialized views in some systems).

▶ Example

CREATE VIEW high_salary_employees AS
SELECT emp_id, emp_name, salary
FROM employees
WHERE salary >= 75000;

SELECT * FROM high_salary_employees;

Output:
2 Ravi  75000
3 Meera 80000
5 Isha  75000

Use cases

  • Hide complex joins behind simple interface.
  • Restrict access to sensitive columns.
  • Encapsulate reusable query logic.

🌟 Interview Tips

  • Not all views are updatable; depends on query complexity and DB engine rules.
  • Differentiate normal view vs materialized view.

▶ Theory Deep Dive

Database engineering decisions should connect schema design, query behavior, indexing, transactions, and scalability constraints.
