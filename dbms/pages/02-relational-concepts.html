Relational Database Concepts

Tables, rows, columns, keys, and constraints that define data correctness.

📚 Core entities

  • Table: relation
  • Row: tuple/record
  • Column: attribute/field

🔑 Keys

  • Primary Key: unique + not null identifier.
  • Foreign Key: references primary key of another table.
  • Candidate Key: minimal unique identifier candidates.
  • Composite Key: key made of multiple columns.
  • Super Key: any set of columns that uniquely identifies rows.
  • Alternate Key: candidate keys not chosen as primary key.

🛡 Constraints

PRIMARY KEY
FOREIGN KEY
UNIQUE
NOT NULL
CHECK
DEFAULT

▶ Example schema

CREATE TABLE departments (
  dept_id INT PRIMARY KEY,
  dept_name VARCHAR(50) UNIQUE NOT NULL
);

CREATE TABLE employees (
  emp_id INT PRIMARY KEY,
  emp_name VARCHAR(50) NOT NULL,
  dept_id INT,
  salary DECIMAL(10,2) CHECK (salary >= 0),
  FOREIGN KEY (dept_id) REFERENCES departments(dept_id)
);

🌟 Interview Tips

  • Primary key can be composite.
  • Foreign key can be NULL unless explicitly restricted.
  • Always explain keys using uniqueness + minimality + referential integrity.
