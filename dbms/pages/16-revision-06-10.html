Revision Chunk B (06-10)

Subqueries, normalization, ACID, indexing, and views.

📚 10-Minute Recall

  • Correlated subquery runs per outer row.
  • Normalization removes anomalies (1NF→BCNF).
  • ACID protects transactional integrity.
  • Indexes speed reads, may slow writes.
  • Views simplify access and can improve security abstraction.

⚠ Common Traps

  • Using correlated subqueries blindly on large tables.
  • Over-normalizing read-heavy reporting workloads.
  • Adding too many indexes on write-heavy tables.
  • Assuming all views are updatable.

🌟 Interview Tip

Explain trade-offs explicitly: consistency vs performance, normalization vs query simplicity, isolation vs concurrency.

▶ Theory Deep Dive

Database engineering decisions should connect schema design, query behavior, indexing, transactions, and scalability constraints.
