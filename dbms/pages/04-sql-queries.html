SQL Queries

SELECT, WHERE, ORDER BY, GROUP BY, HAVING, DISTINCT, LIMIT with sample data and outputs.

📋 Sample tables

employees
+-------+----------+---------+--------+
| emp_id| emp_name | dept_id | salary |
+-------+----------+---------+--------+
| 1     | Asha     | 10      | 60000  |
| 2     | Ravi     | 20      | 75000  |
| 3     | Meera    | 10      | 80000  |
| 4     | Karan    | NULL    | 50000  |
| 5     | Isha     | 20      | 75000  |
+-------+----------+---------+--------+

▶ SELECT + WHERE + ORDER BY

SELECT emp_name, salary
FROM employees
WHERE salary >= 70000
ORDER BY salary DESC;

Output:
+----------+--------+
| emp_name | salary |
+----------+--------+
| Meera    | 80000  |
| Ravi     | 75000  |
| Isha     | 75000  |
+----------+--------+

▶ DISTINCT

SELECT DISTINCT dept_id FROM employees;

Output:
+---------+
| dept_id |
+---------+
| 10      |
| 20      |
| NULL    |
+---------+

▶ GROUP BY + HAVING

SELECT dept_id, COUNT(*) AS emp_count, AVG(salary) AS avg_salary
FROM employees
WHERE dept_id IS NOT NULL
GROUP BY dept_id
HAVING COUNT(*) >= 2;

Output:
+---------+-----------+------------+
| dept_id | emp_count | avg_salary |
+---------+-----------+------------+
| 10      | 2         | 70000      |
| 20      | 2         | 75000      |
+---------+-----------+------------+

▶ LIMIT

SELECT emp_name, salary
FROM employees
ORDER BY salary DESC
LIMIT 2;

Output:
+----------+--------+
| emp_name | salary |
+----------+--------+
| Meera    | 80000  |
| Ravi     | 75000  |
+----------+--------+

🌟 Interview Tips

  • Use WHERE before grouping, HAVING after grouping.
  • DISTINCT removes duplicate result rows, not table rows.
