The Four Pillars of OOP

Encapsulation, Abstraction, Inheritance, and Polymorphism with interview depth.

💡 One-line Analogies

  • Encapsulation: ATM hides internal bank rules.
  • Abstraction: Car steering wheel hides engine mechanics.
  • Inheritance: Child class reuses parent behavior.
  • Polymorphism: Same interface, many implementations.

▶ C++ Pillar Examples

// Encapsulation
class Account {
private: double balance;
public: void deposit(double a){ if (a > 0) balance += a; }
        double getBalance() const { return balance; }
};

// Abstraction
class PaymentGateway { public: virtual void pay(double) = 0; virtual ~PaymentGateway() = default; };
class UpiGateway : public PaymentGateway { public: void pay(double amt) override { /*upi*/ } };

// Inheritance
class Employee { public: virtual double bonus() const { return 1000; } };
class Manager : public Employee { public: double bonus() const override { return 5000; } };

// Polymorphism
void issueBonus(const Employee& e){ cout << e.bonus(); } // dynamic dispatch

▶ Python Pillar Examples

# Encapsulation
class Account:
    def __init__(self): self.__balance = 0
    def deposit(self, amt):
        if amt > 0: self.__balance += amt
    def balance(self): return self.__balance

# Abstraction
from abc import ABC, abstractmethod
class PaymentGateway(ABC):
    @abstractmethod
    def pay(self, amt): ...
class UpiGateway(PaymentGateway):
    def pay(self, amt): pass

# Inheritance + Polymorphism
class Employee:
    def bonus(self): return 1000
class Manager(Employee):
    def bonus(self): return 5000

def issue_bonus(emp: Employee):
    return emp.bonus()

▶ JavaScript Pillar Examples

// Encapsulation with #private fields
class Account {
  #balance = 0;
  deposit(amt){ if (amt > 0) this.#balance += amt; }
  get balance(){ return this.#balance; }
}

// Abstraction via base contract
class PaymentGateway { pay(amt){ throw new Error("Implement pay"); } }
class UpiGateway extends PaymentGateway { pay(amt){ /* upi */ } }

// Inheritance + Polymorphism
class Employee { bonus(){ return 1000; } }
class Manager extends Employee { bonus(){ return 5000; } }
const issueBonus = (emp) => emp.bonus();

📚 Advantages and Disadvantages (Per Pillar)

  • Encapsulation: + protects invariants, reduces coupling; − extra boilerplate.
  • Abstraction: + cleaner APIs and substitution; − over-abstraction can confuse juniors.
  • Inheritance: + reuse and hierarchy modeling; − tight coupling and fragile base-class problems.
  • Polymorphism: + extensibility and OCP-friendly design; − dynamic dispatch can complicate debugging.

🌟 Interview Insights

  • When asked any pillar, answer in this order: definition → analogy → code → trade-off.
  • Mention composition as alternative to inheritance to show mature design judgment.
  • In C++, discuss virtual dispatch and vtable briefly when asked runtime polymorphism.
