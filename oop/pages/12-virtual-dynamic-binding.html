Virtual Functions and Dynamic Binding

Runtime polymorphism and late binding in interview-friendly terms.

📄 Binding Types

Early binding (static): call target known at compile time
Late binding (dynamic): call target chosen at runtime by actual object type

▶ C++ Runtime Polymorphism

class Shape { public: virtual double area() const = 0; virtual ~Shape()=default; };
class Circle : public Shape { double r; public: Circle(double r):r(r){} double area() const override { return 3.14*r*r; }};
class Rect : public Shape { double w,h; public: Rect(double w,double h):w(w),h(h){} double area() const override { return w*h; }};

double totalArea(const vector<Shape*>& shapes){
  double sum = 0;
  for (auto s : shapes) sum += s->area(); // dynamic dispatch
  return sum;
}

⚠ Interview pitfalls

  • Missing virtual destructor in polymorphic base class can cause partial destruction.
  • Object slicing when storing derived objects by value in base containers.
  • Confusing overriding with overloading.

🌟 Interview Insight

Say this clearly: “Runtime polymorphism allows extending behavior without changing caller code.” That directly connects to Open-Closed Principle.
