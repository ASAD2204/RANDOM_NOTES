Inheritance Types

Single, Multiple, Multilevel, Hierarchical, and Hybrid inheritance with practical implications.

📄 Concept Diagram

Single:       A -> B
Multiple:     A, B -> C
Multilevel:   A -> B -> C
Hierarchical: A -> B and A -> C
Hybrid:       Combination (e.g., hierarchical + multiple)

▶ Examples

// C++ Multiple Inheritance
class Printer { public: void print(); };
class Scanner { public: void scan(); };
class MultiFunctionDevice : public Printer, public Scanner {};

# Python Multilevel
class A: pass
class B(A): pass
class C(B): pass

// JavaScript (single inheritance with extends)
class Animal {}
class Dog extends Animal {}
// Multiple inheritance simulated via composition/mixins

⚠ Interview Tricky Points

  • JS has prototypal inheritance with single extends; multiple behavior is typically mixin/composition.
  • C++ multiple inheritance can create ambiguity (same method name from two parents).
  • Hybrid inheritance in C++ may introduce diamond problem; virtual inheritance can resolve shared base duplication.
