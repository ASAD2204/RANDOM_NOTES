Classes and Objects

Definition, structure, attributes, methods, object creation, and lifecycle.

📚 Core Definitions

  • Class: blueprint defining fields + methods.
  • Object: runtime instance of a class.
  • Attributes/Fields: state variables inside object.
  • Methods: behavior functions bound to object.

📄 Structure Diagram

Class Account
 ├─ data: owner, balance
 ├─ constructor(owner, initialBalance)
 ├─ deposit(amount)
 └─ withdraw(amount)

Object a1 = Account("Riya", 5000)
Object a2 = Account("Aman", 2500)

▶ Cross-language quick example

// C++
class Account { public: string owner; double balance; void deposit(double a){ balance += a; } };

# Python
class Account:
    def __init__(self, owner, balance):
        self.owner = owner; self.balance = balance
    def deposit(self, a): self.balance += a

// JavaScript
class Account {
  constructor(owner, balance){ this.owner = owner; this.balance = balance; }
  deposit(a){ this.balance += a; }
}

⚠ Tricky Cases

  • Uninitialized fields in C++ leading to undefined behavior.
  • Mutable default argument bugs in Python constructors.
  • Using arrow functions for prototype methods in JS without understanding this semantics.

🌟 Interview Insights

  • Explain object lifecycle: create → initialize → use → destroy/GC.
  • When asked “Class vs Object”, answer with one concrete example from a domain (e.g., Employee and e1).
  • Mention constructor invariants: valid object should never exist in broken state.
