Common OOP Problems and Edge Cases

Diamond problem, ambiguity, slicing, copy semantics, and memory leaks.

📄 Diamond Problem

   A
  / \
 B   C
  \ /
   D

Without virtual inheritance in C++, D may contain two A subobjects.

▶ Edge Cases

  • Multiple inheritance ambiguity: both parents define same method; call is ambiguous unless qualified.
  • Object slicing: assigning derived object to base by value loses derived-specific state.
  • Deep vs shallow copy: shallow copying owning pointer leads to double-delete.
  • Memory leaks: forgetting virtual destructor or delete for heap allocations.

🌟 Interview Fixes

  • Use virtual inheritance to solve diamond shared-base duplication when needed.
  • Use references/pointers to base to avoid slicing.
  • Use Rule of 3/5 or smart pointers for safe ownership.
  • Prefer RAII in C++; avoid manual resource management.

▶ Theory Deep Dive

Object-oriented design should be explained through abstraction boundaries, coupling/cohesion, and maintainability trade-offs.
