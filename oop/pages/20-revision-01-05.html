Revision Chunk A (01-05)

Introduction, Classes/Objects, Four Pillars, Inheritance Types, Composition vs Inheritance.

📚 10-Minute Recall

  • OOP models software as interacting objects with state + behavior.
  • Class is blueprint; object is runtime instance.
  • Encapsulation protects invariants; abstraction hides complexity.
  • Inheritance models "is-a"; composition models "has-a".
  • Prefer composition by default unless subtype relation is truly stable.

📄 Diagram Snapshot

OOP Core:

Reuse:

Composition = has-a
