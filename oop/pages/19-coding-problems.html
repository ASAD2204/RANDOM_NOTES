Coding Problems for OOP Interviews

Design-focused exercises with constraints and what interviewers evaluate.

▶ Problem Set

  1. Parking Lot System: model slots, vehicles, tickets, pricing strategy.
  2. Library Management: books, members, reservations, due policies.
  3. Food Delivery Mini-Backend: users, orders, restaurant strategy, payment gateway.
  4. Notification System: email/sms/push via interface-based pluggable design.
  5. Chess Piece Model: inheritance or composition trade-offs for movement rules.
  6. Cache with Eviction: LRU cache using class design + encapsulated state.
  7. ATM Machine: account services, authentication, transaction logging, error handling.

📄 Evaluation Rubric

Interviewers check:
- Correct domain modeling (classes, responsibilities)
- Extensibility (new feature without rewriting core)
- Proper use of abstraction/interface
- Clean ownership/lifecycle handling
- Unit-testability and dependency boundaries
- Edge case handling and invariants

🌟 Practice Strategy

  • Start with entities and relationships (UML-ish sketch first).
  • Define interfaces before concrete implementations.
  • Write one happy path + two failure path scenarios.
  • Refactor once to remove duplication and improve SRP.

▶ Theory Deep Dive

Object-oriented design should be explained through abstraction boundaries, coupling/cohesion, and maintainability trade-offs.
