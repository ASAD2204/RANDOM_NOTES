Object Lifecycle and Memory

Stack vs heap objects, ownership, destruction timing, and language differences.

📄 Lifecycle timeline

Create -> Initialize -> Use -> Eligible for destruction -> Destroy/Cleanup

C++ stack object: deterministic destruction at scope end.
C++ heap object: destroy via delete (or smart pointer owner).
Python/JS objects: GC-managed; destruction timing non-deterministic.

▶ C++ Ownership Sketch

void f(){
  Widget a;                  // stack (auto-destroy)
  auto p = make_unique<Widget>(); // heap but auto-owned by unique_ptr
} // a and p's Widget destroyed here

⚠ Common mistakes

  • Raw pointer ownership confusion causes leaks and double frees.
  • Cyclic references in GC languages can delay reclamation.
  • Assuming destruction order incorrectly in inheritance chains.
