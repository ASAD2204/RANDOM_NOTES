Revision Chunk C (11-15)

Static Members, Dynamic Binding, SOLID, Edge Cases, Lifecycle & Memory.

📚 10-Minute Recall

  • Static members are class-level state/behavior; use carefully.
  • Virtual/dynamic binding enables runtime polymorphism.
  • SOLID improves maintainability and testability.
  • Edge cases: diamond, ambiguity, object slicing, shallow copy, leaks.
  • Lifecycle clarity prevents ownership bugs and memory issues.

📄 Must-Remember Trio

LSP: subtype must preserve base contract
DIP: high-level depends on abstraction
RAII: acquire resource in ctor, release in dtor

▶ Rapid Fire Interview Checks

  1. Explain why polymorphic bases need virtual destructors.
  2. Give a concrete LSP violation and fix.
  3. When does static state harm testability?
  4. Object slicing: what is lost and how to avoid it?
  5. How does RAII differ from GC-based cleanup?

▶ Theory Deep Dive

Object-oriented design should be explained through abstraction boundaries, coupling/cohesion, and maintainability trade-offs.
