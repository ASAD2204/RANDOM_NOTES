Abstract Classes and Interfaces

Defining contracts for extensible systems.

📚 Purpose

  • Abstract classes define shared partial implementation.
  • Interfaces define capability contracts with minimal coupling.
  • Enable plugin-like architecture and testability.

▶ Examples

// C++ interface style
class Logger { public: virtual void log(string)=0; virtual ~Logger()=default; };

# Python abstract base
from abc import ABC, abstractmethod
class Logger(ABC):
    @abstractmethod
    def log(self, msg): ...

// JS contract by convention
class Logger { log(msg){ throw new Error("Implement log"); } }

🌟 Interview Insight

Use abstract/interface when clients should depend on behavior contract, not concrete implementation (DIP + OCP).

▶ Theory Deep Dive

Object-oriented design should be explained through abstraction boundaries, coupling/cohesion, and maintainability trade-offs.
