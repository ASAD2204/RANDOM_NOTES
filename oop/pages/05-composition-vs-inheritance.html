Composition vs Inheritance

“Has-a” vs “is-a”: choose composition by default, inheritance when subtype behavior truly fits.

📚 Key Difference

  • Inheritance: Dog is an Animal.
  • Composition: Car has an Engine.
  • Inheritance models specialization; composition models assembly.

▶ Example

// Inheritance
class Bird { public: virtual void fly(){} };
class Sparrow : public Bird {};

// Composition
class Engine { public: void start(){} };
class Car {
  Engine engine;
public:
  void start(){ engine.start(); }
};

🌟 When to use which?

  • Use inheritance when subtype can substitute base safely (LSP).
  • Use composition when behavior may vary at runtime (swap collaborators).
  • Favor composition for flexibility and lower coupling.
