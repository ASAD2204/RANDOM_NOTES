Mock Interview Rounds (OOP)

Practice as if in real placement interviews: concept round, design round, and debugging round.

📚 Round 1: Concept Drill (10 mins)

  • Define all four OOP pillars with one example each.
  • Difference between abstraction and encapsulation with product analogy.
  • Overloading vs overriding (include language caveats for Python/JS).
  • Aggregation vs composition using lifecycle language.
  • Virtual function and dynamic binding in two lines.

▶ Round 2: Design Drill (20 mins)

Prompt: Design a notification platform for Email/SMS/Push with retry policy and logging.

Expected strong answer includes:
- Notification interface (send)
- Concrete providers (Email/SMS/Push)
- Strategy for retry policy
- Dependency injection for logger/provider
- Open-Closed compliant extension for new channel

▶ Round 3: Debug & Refactor Drill (20 mins)

Prompt: You have a God class handling order creation, payment, inventory, and email. Refactor it.

Expected direction:
- Split into cohesive services (OrderService, PaymentService, InventoryService, NotificationService)
- Introduce interfaces for external adapters
- Move orchestration into application layer
- Add tests at service boundaries

⚠ Common Interview Mistakes

  • Talking only theory without showing class-level design.
  • Ignoring trade-offs (e.g., overusing inheritance).
  • No mention of failure cases and invariants.
  • Hard-coding dependencies, making code untestable.

🌟 Scoring Rubric (Self-Evaluation)

5/5: clear modeling, SOLID alignment, edge cases, extensible design
4/5: good design but weaker trade-off discussion
3/5: correct basics but tight coupling and missing abstractions
2/5: mostly procedural code with classes as wrappers
1/5: unclear design and no ownership/contract thinking
