Revision Chunk B (06-10)

Class Relationships, Overloading/Overriding, Constructors/Destructors, Access Modifiers, Abstract/Interface.

📚 10-Minute Recall

  • Association: weak relation; Aggregation: weak ownership; Composition: strong ownership.
  • Overloading = same name, different signature; Overriding = child redefines parent behavior.
  • C++ constructors/destructors are deterministic; GC languages are not.
  • Access modifiers enforce encapsulation boundaries.
  • Abstract class shares base behavior; interface defines pure contract.

⚠ Tricky Cases

  • In C++, missing virtual means hiding instead of overriding.
  • Shallow copy of resource-owning object causes double free.
  • "Everything public" destroys invariants.
  • Fat interfaces violate Interface Segregation Principle.

▶ Rapid Fire Interview Checks

  1. Aggregation vs composition using ownership/lifecycle wording.
  2. Why does overriding need compatible contract (LSP angle)?
  3. When to pick abstract class over interface?
  4. What are constructor invariants?
  5. Why should external code avoid direct mutable field access?

▶ Theory Deep Dive

Object-oriented design should be explained through abstraction boundaries, coupling/cohesion, and maintainability trade-offs.
