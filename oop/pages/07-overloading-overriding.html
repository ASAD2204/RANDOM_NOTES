Method Overloading vs Method Overriding

Compile-time polymorphism vs runtime polymorphism.

📄 Quick Difference

Overloading:
- Same method name, different signatures
- Usually resolved at compile time

Overriding:
- Child redefines parent virtual/overridable method
- Resolved at runtime (dynamic dispatch)

▶ Edge-case Examples

// C++ overloading
int area(int s); int area(int l, int b);

// C++ overriding
class Base{ public: virtual void show(); };
class Child: public Base{ public: void show() override; };

# Python: no true compile-time overloading
# emulate with default args or *args
class A:
    def f(self, x, y=0): return x+y

// JS: no signature-based overloading; manual checks by args.length/types
function area(a,b){ if (b===undefined) return a*a; return a*b; }

⚠ Tricky Cases

  • In C++, missing virtual means method hiding, not overriding.
  • In Python/JS, redefining same function name replaces prior one.
  • In C++, mismatched const/ref qualifiers can accidentally fail override.
