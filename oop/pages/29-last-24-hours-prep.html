📚 Must-Remember One-Liners

  • Encapsulation: protect state and enforce invariants through controlled APIs.
  • Abstraction: expose what to do, hide how it is done.
  • Inheritance: use only when subtype truly satisfies base contract (LSP).
  • Polymorphism: same interface, multiple runtime behaviors.
  • Composition over inheritance: default for flexibility and low coupling.
  • SOLID: maintainability framework, not a strict ritual.

⚠ Red-Flag Edge Cases Interviewers Love

  • Diamond problem and multiple inheritance ambiguity (C++).
  • Object slicing when derived stored by value as base.
  • Missing virtual destructor in polymorphic base class.
  • Shallow copy of owning pointer causing double free/leak.
  • Overusing static globals harming testability.
  • Fat interfaces violating Interface Segregation Principle.
