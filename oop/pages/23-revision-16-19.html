Revision Chunk D (16-19)

Best Practices, Brain Teasers, Interview Questions, Coding Problems.

📚 10-Minute Recall

  • Prefer composition, keep classes cohesive, depend on abstractions.
  • Answer tricky questions using principle + failure mode + mitigation.
  • For interview Q&A, use concise structure and practical examples.
  • For coding rounds, design entities first, then responsibilities, then interfaces.

🌟 Coding Round Checklist

  • Identify entities and relationships before coding.
  • Define invariants and error handling paths.
  • Avoid god classes; enforce SRP.
  • Inject dependencies; avoid hard-coded globals.
  • State extensibility plan (new feature path with minimal change).

▶ Final Mock Questions

  1. Design parking lot and support new vehicle type with minimal code change.
  2. Model payment strategies and show OCP-compliant extension.
  3. Refactor a god class into cohesive components.
  4. Show how interface-based design improves unit testing.

▶ Theory Deep Dive

Object-oriented design should be explained through abstraction boundaries, coupling/cohesion, and maintainability trade-offs.
