Brain Teasers

Conceptual questions that test deep OOP understanding.

▶ Teasers

  1. Can a private method be polymorphic? (Language-dependent subtleties.)
  2. Why should polymorphic base classes have virtual destructors in C++?
  3. When does inheritance violate LSP even if code compiles?
  4. Can composition completely replace inheritance? If not, when?
  5. Why is “has-a” often safer than “is-a” in evolving systems?
  6. How can overusing interfaces increase complexity?
  7. Is object creation itself a design smell in some domains (factory needed)?
  8. What breaks if you expose mutable internal collections directly?

🌟 How to answer

  • Start from principle, then edge case, then production example.
  • Name one failure mode and one mitigation.
  • Use concise language: definition + implication + trade-off.

▶ Theory Deep Dive

Object-oriented design should be explained through abstraction boundaries, coupling/cohesion, and maintainability trade-offs.
