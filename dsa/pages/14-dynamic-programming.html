Dynamic Programming

Memoization + tabulation to solve overlapping subproblems efficiently.

📚 Concept

  • Memoization: top-down recursion + cache.
  • Tabulation: bottom-up table filling.

💡 Analogy

Instead of solving same puzzle repeatedly, write solved subpuzzles on sticky notes and reuse them.

📄 Step-by-step

1) Define state (what variables identify subproblem)
2) Write recurrence transition
3) Set base cases
4) Choose memoization or tabulation
5) Optimize space if possible

▶ Implementations (Fibonacci)

// C++ tabulation
vector dp(n+1); dp[0]=0; dp[1]=1;

# Python memoization
from functools import lru_cache

// JS tabulation
const dp=Array(n+1).fill(0);

⏱ Complexity

Usually O(number of states × transitions). Space O(number of states).

✅ / ❌

  • ✅ Converts exponential recursion to polynomial in many problems.
  • ❌ State design can be tricky; memory-heavy tables possible.

⚠ Common Mistakes

  • Wrong state definition causing missing information.
  • Incorrect transition order in tabulation.
  • Ignoring modulo constraints/overflow in counting problems.

🌟 Interview Tips

  • Say state in plain English first.
  • Classic DP sets: knapsack, LIS, LCS, coin change, house robber, partition.

▶ Theory Deep Dive

Data structure and algorithm choices should be justified by complexity, memory behavior, and workload characteristics.
