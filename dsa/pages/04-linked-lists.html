Linked Lists

Singly, doubly, circular linked lists with operations and interview patterns.

📚 Concept

Linked list stores nodes with pointers/references. Non-contiguous memory; insertion/deletion at known node is O(1).

💡 Analogy

Like a treasure hunt chain: each clue points to next location; no random jump to middle without walking through links.

📄 Step-by-step (delete node by value)

1) Handle empty list
2) If head is target, move head=head.next
3) Traverse until prev.next matches target
4) Bypass target: prev.next = prev.next.next

▶ Implementations

// C++ node
struct Node{int v; Node* next; Node(int x):v(x),next(nullptr){} };

# Python node
class Node:
    def __init__(self,v): self.v=v; self.next=None

// JS node
class Node { constructor(v){ this.v=v; this.next=null; } }

⏱ Complexity

  • Insert/Delete head: O(1)
  • Search/access by index: O(n)
  • Insert after known node: O(1)

✅ / ❌

  • ✅ Flexible size, efficient local insert/delete.
  • ❌ Extra pointer memory, poor cache locality, no O(1) indexing.

⚠ Common Mistakes

  • Losing head pointer during operations.
  • Not handling null edge cases (empty/single node).
  • Memory leaks in manual-memory languages.

🌟 Interview Tips

  • Master fast/slow pointers (cycle detection, middle node).
  • Reversal, merge two lists, remove nth from end are must-practice.
