Strings

Manipulation, matching basics, and high-frequency interview operations.

📚 Concept

String is a sequence of characters. In Python/JS strings are immutable; in C++ std::string is mutable-like container.

💡 Analogy

String is like a sentence strip: reading is easy, but edits in middle often require rebuilding parts.

📄 Step-by-step (reverse words)

1) Trim spaces
2) Split by spaces
3) Reverse token order
4) Join with single spaces

▶ Implementations

// C++
string s="hello"; reverse(s.begin(), s.end());

# Python
s="hello"; rev=s[::-1]

// JS
const s="hello"; const rev=s.split("").reverse().join("");

⏱ Complexity

  • Traversal: O(n)
  • Concatenation in loop (naive): can become O(n²)
  • Pattern search naive: O(n*m)

✅ / ❌

  • ✅ Compact representation, rich library support.
  • ❌ Immutability can increase allocations if misused.

⚠ Common Mistakes

  • Ignoring case normalization in comparisons.
  • Repeated concatenation in loops (prefer builder/join).
  • Not handling Unicode/locale considerations when needed.

🌟 Interview Tips

  • Know palindrome, anagram, frequency map, and substring window problems.
  • State whether solution is in-place or allocates new string.

▶ Theory Deep Dive

Data structure and algorithm choices should be justified by complexity, memory behavior, and workload characteristics.
