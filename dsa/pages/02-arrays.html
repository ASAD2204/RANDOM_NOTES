Arrays

Traversal, insertion, deletion, searching, dynamic arrays, and common interview patterns.

📚 Concept

Array is contiguous memory with O(1) index access. Dynamic arrays resize automatically (vector/list/JS array) with amortized append.

💡 Analogy

Array is a numbered locker row: fast direct access by locker number; insertion in middle shifts many lockers.

📄 Step-by-step Working (insert at index i)

1) Ensure capacity (resize if needed)
2) Shift elements right from end to i
3) Place new value at i
4) Increase size

Cost: O(n) due to shifting

▶ Implementations (C++ / Python / JS)

// C++
vector a={1,2,4}; a.insert(a.begin()+2,3);

# Python
a=[1,2,4]; a.insert(2,3)

// JavaScript
let a=[1,2,4]; a.splice(2,0,3);

⏱ Complexity

  • Access: O(1)
  • Search (unsorted): O(n)
  • Insert/Delete end (dynamic array): amortized O(1)
  • Insert/Delete middle: O(n)

✅ Advantages / ❌ Disadvantages

  • ✅ Cache-friendly, simple, fast indexing.
  • ❌ Costly middle updates, fixed-size limitation for static arrays.

⚠ Common Mistakes

  • Off-by-one boundaries in loops.
  • Confusing capacity with size in dynamic arrays.
  • Using nested loops where hash map can reduce complexity.

🌟 Interview Tips

  • Master patterns: prefix sum, two pointers, sliding window.
  • For in-place operations, mention extra space as O(1).

Data structure and algorithm choices should be justified by complexity, memory behavior, and workload characteristics.
