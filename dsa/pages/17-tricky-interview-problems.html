Tricky Interview Problems

Problems that test deep understanding, pattern switching, and edge-case handling.

▶ Problem Set

  1. Median of two sorted arrays (binary search partition).
  2. LRU cache design (hash map + doubly linked list).
  3. Trapping rain water (two pointers or stack).
  4. Word ladder shortest transformation (BFS).
  5. Serialize/deserialize binary tree.
  6. Find duplicate number without modifying array (Floyd cycle).
  7. Kth smallest in sorted matrix (heap / binary search on answer).

📚 How to approach

  • Identify data structure + pattern before coding.
  • State brute-force first, then optimize.
  • Validate invariants on sample input.

📄 Step-by-step interview flow

1) Clarify constraints
2) Brute-force and complexity
3) Optimized approach with rationale
4) Walk through test case
5) Discuss edge cases

⚠ Common Mistakes

  • Jumping to code without confirming constraints.
  • Pattern mismatch (using DP when greedy works, etc.).
  • Not testing empty/single/duplicate-heavy cases.

🌟 Interview Tips

  • Interviewers reward communication and correction ability.
  • If stuck, simplify problem and solve smaller variant aloud.

▶ Theory Deep Dive

Data structure and algorithm choices should be justified by complexity, memory behavior, and workload characteristics.
