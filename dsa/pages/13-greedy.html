Greedy Algorithms

Make locally optimal choices hoping to reach global optimum.

📚 Concept

Greedy works when problem has greedy-choice property and optimal substructure.

💡 Analogy

Picking highest value-per-weight item first in a fractional knapsack.

📄 Step-by-step

1) Define local best choice criterion
2) Sort/priority structure by criterion
3) Pick best repeatedly
4) Prove or justify optimality condition

▶ Implementations

// C++ activity selection by finish time
sort(a.begin(),a.end(),cmpEnd);

# Python interval scheduling
intervals.sort(key=lambda x:x[1])

// JS coin with sorted denominations (canonical systems)
coins.sort((a,b)=>b-a)

⏱ Complexity

Often dominated by sorting: O(n log n), space O(1)-O(n).

✅ / ❌

  • ✅ Usually simple and fast.
  • ❌ Fails on problems requiring global lookahead.

⚠ Common Mistakes

  • Using greedy without proof/intuition.
  • Confusing 0/1 knapsack (DP) with fractional knapsack (greedy).

🌟 Interview Tips

  • If unsure greedy works, present counterexample or shift to DP.

▶ Theory Deep Dive

Data structure and algorithm choices should be justified by complexity, memory behavior, and workload characteristics.
