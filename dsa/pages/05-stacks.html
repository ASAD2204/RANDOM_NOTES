Stacks

LIFO data structure, implementation options, and applications like recursion and expression evaluation.

📚 Concept

Stack follows Last-In First-Out. Core operations: push, pop, top/peek.

💡 Analogy

Plate stack: latest plate placed on top is removed first.

📄 Step-by-step (balanced parentheses)

1) Traverse chars
2) Push opening brackets
3) On closing bracket, pop and match
4) End with empty stack => valid

▶ Implementations

// C++
stack st; st.push(1); st.pop();

# Python
st=[]; st.append(1); st.pop()

// JS
const st=[]; st.push(1); st.pop();

⏱ Complexity

  • Push/Pop/Top: O(1)
  • Space: O(n)

✅ / ❌

  • ✅ Simple and fast.
  • ❌ Restricted access only at top.

⚠ Common Mistakes

  • Popping empty stack without checks.
  • Using recursion without stack-depth awareness.
  • Forgetting stack reset across test cases.

🌟 Interview Tips

  • Know monotonic stack for next greater/smaller element patterns.
  • Use stack for DFS iterative and parsing problems.

▶ Theory Deep Dive

Data structure and algorithm choices should be justified by complexity, memory behavior, and workload characteristics.
