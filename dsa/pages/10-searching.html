Searching Algorithms

Linear search and binary search with clear use-cases and complexity trade-offs.

📚 Concept

  • Linear search scans each element; works on any array/list.
  • Binary search halves search space; requires sorted data.

💡 Analogy

Linear: finding name by reading every page. Binary: opening dictionary at middle repeatedly.

📄 Step-by-step (binary search)

1) l=0, r=n-1
2) m=l+(r-l)/2
3) if a[m]==x return m
4) if a[m] < x, l=m+1 else r=m-1
5) repeat until l>r

⏱ Complexity

  • Linear: O(n) time, O(1) space
  • Binary: O(log n) time, O(1) iterative space

✅ / ❌

  • ✅ Binary is very fast on sorted data.
  • ❌ Sorting first can dominate cost if one-time search only.

⚠ Common Mistakes

  • Applying binary search on unsorted array.
  • Infinite loop due to wrong mid update.
  • Overflow with (l+r)/2 in fixed-width ints.

🌟 Interview Tips

  • Use lower_bound / upper_bound style questions frequently appear.
  • State “binary search on answer” for monotonic feasibility problems.
