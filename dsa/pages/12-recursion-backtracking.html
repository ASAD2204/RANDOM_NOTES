Recursion and Backtracking

Recursive thinking, state-space exploration, and constraint-based undo patterns.

📚 Concept

  • Recursion: function calls itself on smaller subproblem.
  • Backtracking: DFS over choices with “choose → explore → unchoose”.

💡 Analogy

Maze solving: try a path; if blocked, return to previous junction and try another route.

📄 Step-by-step template

def backtrack(state):
  if goal(state): save
  for choice in choices(state):
    apply(choice)
    backtrack(state)
    undo(choice)

⏱ Complexity

  • Depends on branching factor and depth (often exponential).
  • Space includes recursion stack O(depth).

✅ / ❌

  • ✅ Elegant for combinatorial search.
  • ❌ Can blow up exponentially without pruning.

⚠ Common Mistakes

  • Missing base case causing infinite recursion.
  • Not undoing state in backtracking.
  • Global state bugs across test cases.

🌟 Interview Tips

  • Always explain recursion tree and pruning strategy.
  • Mention memoization if subproblems overlap.
