Visual Cheatsheet

Fast mental diagrams for whiteboard interviews.

📄 Pattern Map

Array/String  -> Two pointers / Sliding window
Linked List   -> Fast-slow pointers
Tree          -> DFS/BFS traversals
Graph         -> BFS/DFS/Dijkstra
Intervals     -> Sort + merge/greedy
DP            -> State + transition table
Backtracking  -> Decision tree + pruning

📄 Complexity Ladder

Best to avoid in interviews when possible:
O(1) < O(log n) < O(n) < O(n log n) < O(n²) < O(2^n)

🌟 Use in interview

Start by naming pattern and target complexity; this shows senior problem-solving maturity before coding.

Data structure and algorithm choices should be justified by complexity, memory behavior, and workload characteristics.
