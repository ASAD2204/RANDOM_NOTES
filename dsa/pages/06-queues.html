Queues

FIFO structure: simple queue, circular queue, priority queue, and deque.

📚 Concept

Queue follows First-In First-Out. Variants support specialized access patterns.

💡 Analogy

Ticket line: first person entering line gets served first.

📄 Step-by-step (circular queue enqueue)

1) Check full: (rear+1)%cap == front
2) If empty initialize front=0
3) rear=(rear+1)%cap
4) arr[rear]=value

▶ Implementations

// C++
queue q; q.push(10); q.pop();
priority_queue pq; pq.push(5);

# Python
from collections import deque
q=deque(); q.append(10); q.popleft()
import heapq
h=[]; heapq.heappush(h,5)

// JS
const q=[]; q.push(10); q.shift();
// Deque efficiently via linked list libs in production

⏱ Complexity

  • Queue enqueue/dequeue: O(1)
  • Priority queue push/pop: O(log n)
  • Deque end operations: O(1)

✅ / ❌

  • ✅ Ideal for scheduling, BFS, buffering.
  • ❌ Restricted access in plain queue, shift-cost pitfalls in naive JS array queue.

⚠ Common Mistakes

  • Using static array queue without circular logic.
  • Using JS shift() for huge queues (can be O(n)).
  • Confusing min-heap vs max-heap behavior in language libraries.

🌟 Interview Tips

  • BFS = queue by default.
  • Priority queue solves top-k and shortest path optimizations.

▶ Theory Deep Dive

Data structure and algorithm choices should be justified by complexity, memory behavior, and workload characteristics.
