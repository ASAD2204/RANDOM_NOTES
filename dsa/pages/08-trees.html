Trees

Binary trees, BST, traversals, AVL, heaps, and tries with interview intuition.

📚 Concept

  • Tree is hierarchical non-linear structure with root and children.
  • BST: left < root < right.
  • AVL: self-balancing BST (height diff <= 1).
  • Heap: complete tree with heap property (min/max).
  • Trie: prefix tree for strings.

💡 Analogy

Organization chart: root CEO, children managers; traversals are different ways to inspect hierarchy.

📄 Step-by-step (inorder traversal)

inorder(node):
  if node is null return
  inorder(node.left)
  visit(node)
  inorder(node.right)

For BST, inorder gives sorted order.

▶ Implementations

// C++ recursive inorder
void inorder(Node* r){ if(!r) return; inorder(r->l); cout<v; inorder(r->r); }

⏱ Complexity

  • Traversal: O(n)
  • BST search insert delete average: O(log n), worst O(n)
  • AVL operations: O(log n)
  • Heap push/pop: O(log n), peek O(1)
  • Trie insert/search: O(L) where L is word length

✅ / ❌

  • ✅ Efficient hierarchical operations and ordered queries.
  • ❌ Implementation complexity for balanced trees/tries.

⚠ Common Mistakes

  • Confusing tree depth vs height.
  • Missing base cases in recursive traversals.
  • Not handling skewed BST worst-case.

🌟 Interview Tips

  • Master preorder/inorder/postorder + level order.
  • For BST questions always mention sorted inorder property.
