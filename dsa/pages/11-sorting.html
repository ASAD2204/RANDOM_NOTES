Sorting Algorithms

Bubble, selection, insertion, merge, quick, counting, radix, and heap sort overview.

📚 Concept

Sorting arranges data for efficient search, grouping, and order-based logic.

📄 Step-by-step (merge sort)

1) Divide array into halves recursively
2) Sort each half
3) Merge sorted halves in linear time

Recurrence: T(n)=2T(n/2)+O(n) => O(n log n)

▶ Implementations

// C++
sort(a.begin(), a.end()); // introsort

# Python
a.sort()       # Timsort

// JS
a.sort((x,y)=>x-y);

⏱ Complexity Table (important)

  • Bubble/Selection/Insertion: O(n²)
  • Merge: O(n log n), space O(n)
  • Quick: average O(n log n), worst O(n²)
  • Heap sort: O(n log n), space O(1)
  • Counting: O(n+k), Radix: O(d*(n+k))

✅ / ❌

  • ✅ Comparison sorts are general-purpose.
  • ❌ Non-comparison sorts need bounded/integer-like key assumptions.

⚠ Common Mistakes

  • Forgetting stable vs unstable behavior requirements.
  • Not passing numeric comparator in JS.
  • Using counting sort for huge key ranges.

🌟 Interview Tips

  • Explain why O(n log n) lower bound applies to comparison sorting.
  • Mention Timsort (Python) and introsort (C++) as practical defaults.

▶ Theory Deep Dive

Data structure and algorithm choices should be justified by complexity, memory behavior, and workload characteristics.
