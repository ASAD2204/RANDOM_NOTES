Graphs

Representations, BFS/DFS, shortest path intuition, and real applications.

📚 Concept

Graph = vertices + edges (directed/undirected, weighted/unweighted).

💡 Analogy

City map: intersections are nodes, roads are edges; shortest route problem is shortest path.

📄 Step-by-step (BFS)

1) Push start into queue
2) Mark visited
3) Pop node, process it
4) Push unvisited neighbors
5) Repeat until queue empty

▶ Implementations

// C++ adjacency list
vector> g(n);

# Python BFS
from collections import deque
q=deque([s]); vis={s}

// JS DFS iterative
const st=[s], vis=new Set([s]);

⏱ Complexity

  • BFS/DFS: O(V+E)
  • Dijkstra (heap): O((V+E)logV)
  • Space: O(V+E)

✅ / ❌

  • ✅ Models networks and dependencies naturally.
  • ❌ Can be heavy in memory for dense graphs.

⚠ Common Mistakes

  • Forgetting visited set leads to cycles/infinite loops.
  • Using DFS where shortest path in unweighted graph needs BFS.
  • Mixing 0-index and 1-index node labels.

🌟 Interview Tips

  • State chosen representation (adjacency list vs matrix) and why.
  • For shortest path: BFS (unweighted), Dijkstra (non-negative weighted), Bellman-Ford (negative edges).

▶ Theory Deep Dive

Data structure and algorithm choices should be justified by complexity, memory behavior, and workload characteristics.
