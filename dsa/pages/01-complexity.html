Algorithm Complexity Analysis

Time complexity, space complexity, Big O, Big Theta, and Big Omega for interview decision-making.

📚 Concept

  • Time complexity: growth of operations with input size n.
  • Space complexity: extra memory used by algorithm.
  • Big O: upper bound (worst-case growth).
  • Big Theta: tight bound (exact asymptotic order).
  • Big Omega: lower bound (best-case growth).

💡 Analogy

Choosing an algorithm is like choosing a commute route: O(1) is elevator, O(log n) is express train, O(n²) is city traffic with two bottlenecks.

📄 Step-by-step Working

1) Identify primitive operation (comparison / assignment)
2) Count how many times it runs
3) Keep dominant term only
4) Drop constants
5) State time + space

Example:
for i in [0..n-1]:          -> n
  for j in [0..n-1]:        -> n * n
Total: O(n²)

▶ Implementation Examples (conceptual)

// C++: O(log n) binary search idea
int l=0,r=n-1; while(l<=r){ int m=(l+r)/2; ... }

# Python: O(n) linear scan
for x in arr:
    if x == target: return True

// JS: O(n log n) sort baseline
arr.sort((a,b)=>a-b);

⏱ Complexity

  • O(1): hash lookup average
  • O(log n): binary search
  • O(n): traversal
  • O(n log n): merge/heap/quick average sort
  • O(n²): nested loop pair checks

⚠ Common Mistakes

  • Confusing worst-case O with exact runtime.
  • Ignoring auxiliary space in recursion and data structures.
  • Claiming hash map always O(1) (collisions can degrade).

🌟 Interview Tips

  • Always state both time and space after coding.
  • Mention trade-off explicitly: “I used O(n) extra space to reduce time from O(n²) to O(n).”
  • Use Big Theta when asked for tight bound.
