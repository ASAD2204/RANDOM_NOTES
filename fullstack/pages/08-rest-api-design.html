REST API Design

Designing predictable, scalable APIs for frontend and third-party clients.

▶ REST principles

  • Resource-oriented URLs
  • Stateless communication
  • Standard HTTP methods and status codes
  • Consistent request/response formats (usually JSON)

▶ Method + status mapping

GET /users        -> 200
POST /users       -> 201
GET /users/:id    -> 200/404
PUT /users/:id    -> 200/204
DELETE /users/:id -> 204

▶ Versioning and response shape

/api/v1/users
{
  "data": {...},
  "meta": {"requestId":"..."},
  "error": null
}

▶ Theory Deep Dive

REST quality depends on consistent resource modeling, status semantics, idempotency, and versioning strategy aligned with client evolution.
