Web Fundamentals

Core internet and browser concepts for full stack engineering.

▶ How the internet works

Clients and servers communicate over TCP/IP. Data is split into packets and routed through networks to destination hosts.

▶ Client-server + request-response lifecycle

Browser -> DNS lookup -> TCP/TLS handshake -> HTTP request
Server -> App logic + DB query -> HTTP response -> Browser render

▶ HTTP and HTTPS

  • HTTP is stateless protocol for web communication.
  • HTTPS = HTTP over TLS encryption for confidentiality and integrity.
  • Methods indicate action: GET, POST, PUT, DELETE.

▶ DNS and browsers

DNS maps domain names to IP addresses. Browser rendering engine parses HTML/CSS, executes JS, then paints pixels on screen.

▶ Theory Deep Dive

Web applications operate across network, protocol, and rendering layers. Strong engineering decisions require understanding latency sources, caching semantics, and browser-server contract boundaries.
