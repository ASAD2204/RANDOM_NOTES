Authentication and Security

Protecting users, sessions, APIs, and data in full stack systems.

▶ JWT vs session auth

  • JWT: stateless token-based auth, common for SPA/mobile APIs.
  • Session-based: server-side session store + cookie, easier revocation.

▶ Password hashing + middleware

import bcrypt from "bcrypt";
const hash = await bcrypt.hash(password, 12);
const ok = await bcrypt.compare(inputPassword, hash);
function requireAuth(req, res, next){
  const token = req.headers.authorization?.split(" ")[1];
  if(!token) return res.status(401).json({message:"Unauthorized"});
  next();
}

🌟 API security checklist

  • HTTPS everywhere
  • Rate limiting + input validation
  • CORS policy hardening
  • Least-privilege authorization
  • Secure headers and secret management

▶ Theory Deep Dive

Application security is a layered model: authentication, authorization, transport security, secret hygiene, and abuse prevention must all be designed together.
