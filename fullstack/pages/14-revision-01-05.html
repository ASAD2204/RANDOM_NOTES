Revision 01-05 (Web to Express)

Fast recap for frontend + Node/Express core.

▶ Must-remember points

  • Request lifecycle: DNS -> TCP/TLS -> HTTP -> render.
  • JS async model: event loop, microtasks, async/await.
  • React: state-driven UI, effects for side effects, props for composition.
  • Node: non-blocking I/O and event-driven design.
  • Express: middleware pipeline, routing, centralized error handling.

🌟 60-second explanation

Frontend triggers API calls; backend validates and serves data;
database persists state; auth + monitoring secure operations.

▶ Theory Deep Dive

These topics form the execution backbone of web apps: transport model, JS runtime behavior, frontend state model, and backend request handling.
