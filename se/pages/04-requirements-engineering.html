Requirements Engineering

Gathering, analysis, validation, and documentation of software requirements.

📚 Stages

  • Requirement gathering (interviews, workshops, surveys, observation)
  • Analysis and prioritization
  • Validation (review, prototype, testability checks)
  • Documentation (SRS/user stories/use cases)

🧾 Requirement types

  • Functional: what system should do.
  • Non-functional: performance, security, scalability, availability.

▶ Practical example

Feature: User login
Functional: user can login via email+password
Non-functional: response under 300ms for 95% requests
Security: account lock after 5 failed attempts

🌟 Interview Tips

  • Ambiguous requirement is a risk; ask clarifying questions early.
  • Good requirements are specific, measurable, testable, and traceable.

▶ Theory Deep Dive

Software engineering concepts should be framed as process-quality trade-offs across requirements, design, testing, delivery, and maintenance.
