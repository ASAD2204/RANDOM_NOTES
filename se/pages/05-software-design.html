Software Design

Modularity, coupling, cohesion, and UML design communication.

📚 Core principles

  • Modular design for separation of concerns
  • Low coupling between modules
  • High cohesion within module
  • Clear interfaces and contracts

UML diagrams (interview level)

Class diagram: static structure (classes, fields, methods, relationships)
Sequence diagram: runtime interaction order between objects/services

▶ Example

OrderService -> PaymentService -> InventoryService -> NotificationService

Good design:
- each service has clear responsibility
- dependencies via interfaces
- easier testing and replacement

🌟 Interview Tips

  • When discussing design, mention maintainability and testability impact.
  • High cohesion + low coupling is a standard design quality answer.

▶ Theory Deep Dive

Software engineering concepts should be framed as process-quality trade-offs across requirements, design, testing, delivery, and maintenance.
