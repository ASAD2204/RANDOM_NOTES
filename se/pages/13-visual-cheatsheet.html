Software Engineering Visual Cheatsheet

One-page mental map for quick recall before interviews.

▶ Lifecycle map

Problem -> Requirements -> Design -> Build -> Test -> Deploy -> Maintain

▶ Model selection thumb rule

  • Stable + regulated: Waterfall/V-Model
  • Changing requirements: Agile/Scrum
  • High risk/unknowns: Spiral

▶ Quality + risk reminders

Quality = process discipline + testing depth + feedback speed
Risk = probability × impact; reduce via mitigation and monitoring

▶ Theory Deep Dive

Software engineering concepts should be framed as process-quality trade-offs across requirements, design, testing, delivery, and maintenance.
