Software Maintenance

Post-release evolution: fixing, adapting, and improving software over time.

📚 Maintenance types

  • Corrective: bug fixes after release
  • Adaptive: changes due to environment/platform updates
  • Perfective: feature or performance improvements
  • Preventive: refactoring to reduce future issues

▶ Practical examples

  • Corrective: fix crash in payment flow.
  • Adaptive: update app for new OS/browser API.
  • Perfective: optimize report generation from 12s to 2s.
  • Preventive: refactor legacy module to reduce defect rate.

🌟 Interview Tips

  • Most real project cost appears in maintenance, not initial coding.
  • Good documentation and test coverage reduce maintenance effort.

▶ Theory Deep Dive

Software engineering concepts should be framed as process-quality trade-offs across requirements, design, testing, delivery, and maintenance.
