Software Development Models

Waterfall, Agile, Scrum, Spiral, and Extreme Programming with pros/cons.

📄 Model snapshot

Waterfall: sequential, phase-locked
Agile: iterative + incremental
Scrum: Agile framework with sprints and roles
Spiral: risk-driven iterative model
XP: Agile model focused on engineering practices

Waterfall

  • ✅ Clear documentation and phase control
  • ❌ Poor flexibility for changing requirements

Agile

  • ✅ Fast feedback and adaptability
  • ❌ Scope drift risk if priorities are unstable

Scrum

  • ✅ Structured Agile with sprint cadence
  • ❌ Requires disciplined team collaboration

Spiral

  • ✅ Strong risk management for high-uncertainty projects
  • ❌ Can be expensive and process-heavy

Extreme Programming (XP)

  • ✅ High code quality via TDD, pair programming, CI
  • ❌ Demands strong engineering maturity

🌟 Interview Tips

  • Pick model based on requirement stability, risk, compliance, and team maturity.
  • Differentiate Agile philosophy vs Scrum framework.
