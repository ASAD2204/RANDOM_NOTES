Software Testing

Testing layers and quality assurance strategy for reliable software releases.

📚 Test types

  • Unit Testing: individual function/class behavior
  • Integration Testing: interactions between modules/services
  • System Testing: complete end-to-end system verification
  • Acceptance Testing: business/user validation of requirements

▶ Practical perspective

Test Pyramid:
- Many unit tests
- Fewer integration tests
- Limited end-to-end tests

Goal: fast feedback + confidence

🌟 Interview Tips

  • Differentiate verification (building right?) vs validation (building right product?).
  • Mention automation in CI/CD pipelines for regression control.

▶ Theory Deep Dive

Software engineering concepts should be framed as process-quality trade-offs across requirements, design, testing, delivery, and maintenance.
