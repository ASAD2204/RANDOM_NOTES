📄 Problem 1: FizzBuzz

#include <iostream>
#include <string>
using namespace std;

string fizzBuzz(int n) {
    if (n % 15 == 0) return "FizzBuzz";
    if (n % 3 == 0)  return "Fizz";
    if (n % 5 == 0)  return "Buzz";
    return to_string(n);
}

int main() {
    for (int i = 1; i <= 20; i++)
        cout << fizzBuzz(i) << "\n";
}

▶ Problems 2 – 5

#include <vector>
#include <unordered_map>
#include <string>
#include <algorithm>
using namespace std;

// Problem 2: Two Sum — O(N)
vector<int> twoSum(vector<int>& nums, int target) {
    unordered_map<int,int> seen;
    for (int i = 0; i < (int)nums.size(); i++) {
        int comp = target - nums[i];
        if (seen.count(comp)) return {seen[comp], i};
        seen[nums[i]] = i;
    }
    return {};
}

// Problem 3: Reverse String — O(N), O(1) extra
void reverseString(string& s) {
    int l = 0, r = (int)s.size() - 1;
    while (l < r) swap(s[l++], s[r--]);
}

// Problem 4: Check Palindrome (ignore spaces/case)
bool isPalindrome(string s) {
    string clean;
    for (char c : s)
        if (isalnum(c)) clean += tolower(c);
    int l = 0, r = (int)clean.size() - 1;
    while (l < r) if (clean[l++] != clean[r--]) return false;
    return true;
}

// Problem 5: Frequency Count
unordered_map<char,int> charFreq(const string& s) {
    unordered_map<char,int> freq;
    for (char c : s) freq[c]++;
    return freq;
}

💡 Interview Strategy

Before coding: state the problem in your own words → clarify edge cases (empty input? negatives? case-sensitive?) → state your time/space complexity → then write clean code with meaningful variable names. Interviewers reward this process as much as the code itself.

⚠ Common Mistakes

  • FizzBuzz: checking n % 3 before n % 15 — correct order is divisible-by-15 first.
  • Two Sum: using nested loops O(N²) when hash map O(N) is expected.
  • Palindrome: forgetting to handle spaces, punctuation, and mixed case.

🌟 Interview Tips

  • FizzBuzz is often a warmup — write it in under 60 seconds with the correct priority order.
  • Two Sum is a gateway for hash map discussion — be ready to explain why you use a map.
  • Always test with edge cases out loud: empty string, single char, all same chars.

▶ Problems 2 – 5

from collections import Counter

# Problem 2: Two Sum — O(N)
def two_sum(nums, target):
    seen = {}  # value -> index
    for i, n in enumerate(nums):
        comp = target - n
        if comp in seen:
            return [seen[comp], i]
        seen[n] = i

# Problem 3: Reverse String — two approaches
def reverse_string_slice(s):
    return s[::-1]  # creates new string

def reverse_in_place(s):
    lst = list(s)
    l, r = 0, len(lst) - 1
    while l < r:
        lst[l], lst[r] = lst[r], lst[l]
        l += 1; r -= 1
    return "".join(lst)

# Problem 4: Check Palindrome
def is_palindrome(s):
    cleaned = "".join(c.lower() for c in s if c.isalnum())
    return cleaned == cleaned[::-1]

# Problem 5: Frequency Count
def char_freq(s):
    return Counter(s)                   # one-liner
    # or: {c: s.count(c) for c in set(s)} but O(N²)

print(two_sum([2,7,11,15], 9))         # [0, 1]
print(is_palindrome("A man a plan a canal Panama"))  # True

💡 Interview Strategy

Python's concise syntax can be a double-edged sword: s[::-1] reverses in one line, but always explain what it does. Interviewers prefer you explain your thinking over magical one-liners without justification. Offer both the concise and the explicit version.

⚠ Common Mistakes

  • Forgetting that strings are immutable — s[0] = 'x' raises a TypeError.
  • Using s.count(c) for c in set(s) — O(N²) when Counter is O(N).
  • Palindrome check: not lowercasing or not filtering non-alphanumeric chars.

🌟 Interview Tips

  • Always mention Counter for frequency problems — shows you know the standard library.
  • For Two Sum: walk through the hash map approach step by step — draw it on a whiteboard if available.
  • Python's enumerate is preferred over manual index tracking in for loops.

▶ Problems 2 – 5

// Problem 2: Two Sum — O(N)
function twoSum(nums, target) {
    const seen = new Map();  // value -> index
    for (let i = 0; i < nums.length; i++) {
        const comp = target - nums[i];
        if (seen.has(comp)) return [seen.get(comp), i];
        seen.set(nums[i], i);
    }
    return [];
}

// Problem 3: Reverse String
const reverseString = s => s.split("").reverse().join(""); // O(N)

// Two-pointer from scratch
function reverseTwoPointer(s) {
    const a = s.split("");
    let l = 0, r = a.length - 1;
    while (l < r) {
        [a[l], a[r]] = [a[r], a[l]];
        l++; r--;
    }
    return a.join("");
}

// Problem 4: Palindrome (ignore non-alphanumeric)
function isPalindrome(s) {
    const clean = s.toLowerCase().replace(/[^a-z0-9]/g, "");
    let l = 0, r = clean.length - 1;
    while (l < r) if (clean[l++] !== clean[r--]) return false;
    return true;
}

// Problem 5: Frequency Count
function charFreq(s) {
    const freq = new Map();
    for (const c of s) freq.set(c, (freq.get(c) || 0) + 1);
    return freq;
}

// Tests
console.log(twoSum([2,7,11,15], 9));          // [0, 1]
console.log(isPalindrome("A man a plan a canal Panama")); // true

💡 Interview Strategy

In JavaScript interviews, show functional style (map/filter/reduce) when appropriate, but also demonstrate you can think iteratively. Interviewers test both fluency with modern JS and understanding of fundamentals. Don't rely solely on built-in methods without explaining the complexity.

⚠ Common Mistakes

  • Forgetting .split("").reverse().join("") — strings have no .reverse() method directly.
  • Using an object {} as a frequency map instead of new Map() — prototype keys can conflict.
  • Palindrome: forgetting regex to strip non-alphanumeric chars before checking.

🌟 Interview Tips

  • Two Sum with Map: always mention that Map is O(1) average for get/set vs object O(1) with caveats.
  • For frequency count, offer both Map and object approaches — show versatility.
  • Regex in palindrome: /[^a-z0-9]/g — know how to read and write basic regex patterns.
