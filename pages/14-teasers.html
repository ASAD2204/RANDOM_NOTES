Brain Teasers

Clever bit tricks, swap without temp, power-of-two checks — compact solutions that impress interviewers.

📚 Simple Explanation

  • C++ bit manipulation is extremely fast — direct hardware operations.
  • XOR (^): toggles bits; a ^ a = 0, a ^ 0 = a.
  • n & (n-1): clears the lowest set bit — used to check power of 2.
  • n & (-n): isolates the lowest set bit.

📄 Key Tricks

// Swap without temp variable (XOR trick)
a ^= b;  b ^= a;  a ^= b;

// Check if power of 2
bool isPow2(int n) { return n > 0 && (n & (n-1)) == 0; }

// Count set bits (Brian Kernighan's algorithm)
int countBits(int n) {
    int count = 0;
    while (n) { n &= (n-1); count++; }
    return count;
}

// Check if odd/even without modulo
bool isOdd(int n) { return n & 1; }

▶ Example Code — 5 Teasers with Solutions

#include <iostream>
using namespace std;

// 1. Find the missing number 1..n using XOR
int missingNumber(vector<int>& nums, int n) {
    int xorAll = 0, xorArr = 0;
    for (int i = 1; i <= n; i++) xorAll ^= i;
    for (int x : nums) xorArr ^= x;
    return xorAll ^ xorArr;  // missing number
}

// 2. Reverse bits (32-bit)
uint32_t reverseBits(uint32_t n) {
    uint32_t res = 0;
    for (int i = 0; i < 32; i++) {
        res = (res << 1) | (n & 1);
        n >>= 1;
    }
    return res;
}

// 3. Find single non-duplicate (XOR cancels pairs)
int singleNumber(vector<int>& nums) {
    int res = 0;
    for (int n : nums) res ^= n;
    return res;
}

💡 Real-World Analogy

XOR is like a light switch: flip it twice and you're back to the start. XOR-ing every number in a paired list cancels them out — only the lonely number remains, like the last person without a dance partner.

⚠ Common Mistakes

  • XOR swap only works when a and b are different variables — a ^= a destroys the value if same address.
  • Forgetting that bit operations work on the two's complement representation for negatives.
  • Using bit tricks to avoid clarity — only use them when performance matters or the trick is well-known.

🌟 Interview Tips

  • Explain why a trick works, not just what it does — walkthrough XOR on paper if asked.
  • n & (n-1) clears the lowest set bit: know this for "count set bits" and "power of two".
  • Brian Kernighan's bit-counting algorithm runs in O(number of set bits) — faster than O(32) naive check.

📚 Simple Explanation

  • Python integers are arbitrary precision — no overflow.
  • Bit ops: &, |, ^, ~, <<, >>.
  • Python has unique syntax tricks: tuple swap, chained comparisons, conditional expressions.

fib = lambda n: n if n <= 1 else fib(n-1) + fib(n-2)

# 2. Flatten nested list
def flatten(lst):
    return [x for sub in lst for x in (flatten(sub) if isinstance(sub, list) else [sub])]

# 3. Palindrome check (no extra space conceptually)
def is_palindrome(s):
    s = s.lower().replace(" ", "")
    return s == s[::-1]

# 4. Find missing number 1..n
def missing(nums, n):
    return n*(n+1)//2 - sum(nums)  # no XOR needed in Python!

# 5. Rotate array k steps
def rotate(nums, k):
    k %= len(nums)
    nums[:] = nums[-k:] + nums[:-k]

💡 Real-World Analogy

Python's a, b = b, a is the magic trick version of the XOR swap — it looks simple but Python creates a temporary tuple under the hood. The audience sees clean magic; behind the curtain it still follows the same physics.

⚠ Common Mistakes

  • Recursive lambda for fibonacci — Python has a ~1000 recursion limit; memoize or use iterative.
  • n.bit_count() is only in Python 3.10+ — use bin(n).count('1') for compatibility.
  • Rotate: nums[-0:] when k=0 → k%n=0 returns the whole list — handle edge case.

🌟 Interview Tips

  • For missing number: math formula n(n+1)/2 - sum is cleaner than XOR in Python — mention both approaches.
  • Tuple swap is O(1) — Python creates a temporary tuple then unpacks it.
  • Bit operations in Python work but are rarely the "clever" solution since Python has arithmetic shortcuts.

📚 Simple Explanation

  • JS bitwise ops work on 32-bit signed integers (numbers converted first).
  • Destructuring swap: [a, b] = [b, a].
  • Fun one-liners using reduce, filter, spread, and short-circuit evaluation.

📄 Key Tricks

// Destructuring swap
[a, b] = [b, a];

// Power of 2 check
const isPow2 = n => n > 0 && (n & (n-1)) === 0;

// Single non-duplicate
const single = nums.reduce((acc, n) => acc ^ n, 0);

// Quick integer floor (bitwise OR with 0)
const floor = n => n | 0;    // 3.7 | 0 = 3 (NOT for negatives)
const floor2 = Math.trunc(n); // safer alternative

// Count set bits
const countBits = n => n.toString(2).split("0").join("").length;

▶ Example Code — 5 Teasers with Solutions

// 1. Fibonacci with memoization
const memo = {};
const fib = n => {
    if (n <= 1) return n;
    return memo[n] ?? (memo[n] = fib(n-1) + fib(n-2));
};

// 2. Palindrome check
const isPalindrome = s =>
    s.toLowerCase().replace(/[^a-z0-9]/g, "") ===
    s.toLowerCase().replace(/[^a-z0-9]/g, "").split("").reverse().join("");

// 3. Missing number (math formula)
const missingNum = (nums, n) => n*(n+1)/2 - nums.reduce((a,b) => a+b, 0);

// 4. Rotate array k steps (in-place)
const rotate = (nums, k) => {
    k %= nums.length;
    nums.unshift(...nums.splice(-k)); // elegant but O(N) extra
};

// 5. Flatten with flat()
[1, [2, [3, [4]]]].flat(Infinity);  // [1,2,3,4]

💡 Real-World Analogy

JS ?? (nullish coalescing) as a memoization guard is like a smart cache desk clerk: if the answer is already in the drawer, hand it over immediately; otherwise solve it, file it, and hand it over — never solve twice.

⚠ Common Mistakes

  • n | 0 for flooring only works correctly for positive numbers — use Math.floor for negatives.
  • Bitwise XOR (^) is often confused with exponentiation — JS uses ** for powers, not ^.
  • nums.unshift(...nums.splice(-k)) for rotate — spread crashes on very large arrays (stack limit).

🌟 Interview Tips

  • Destructuring swap [a,b]=[b,a] is the JS "no temp variable" answer — know it cold.
  • .flat(Infinity) is the modern interview answer for deep flatten.
  • XOR reduce for single non-duplicate: nums.reduce((a,b) => a^b, 0) — elegant one-liner.
