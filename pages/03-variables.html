Variables & Data Types

How data is stored, typed, and converted — foundational knowledge for every coding problem.

📚 Simple Explanation

C++ is statically typed: the compiler must know every variable's type at compile time. Primitives have fixed sizes; non-primitives are user-defined structures.

  • Primitives: int, long long, float, double, char, bool
  • Non-primitives: arrays, pointers, structs, classes
  • Type conversion: widening (safe), narrowing (may lose data)

📄 Syntax Snapshot

type varName = value;          // declaration
type varName;                  // uninitialized (dangerous!)
auto x = 42;                   // compiler deduces type (C++11)
int y = static_cast<int>(3.9); // explicit cast

▶ Example Code

#include <iostream>
#include <string>

int main() {
    int    age   = 21;
    double gpa   = 9.1;
    char   grade = 'A';
    bool   pass  = true;
    std::string name = "Alice";

    // Widening: int -> double (safe)
    double d = age;

    // Narrowing: double -> int (truncates)
    int truncated = static_cast<int>(gpa); // 9

    std::cout << truncated << "\n"; // 9
    return 0;
}

💡 Real-World Analogy

Like labeled jars in a kitchen: the jar marked "sugar" can only hold sugar (int jar holds integers). Pouring liquid into it (narrowing cast) might spill some data.

⚠ Common Mistakes

  • 5 / 2 = 2 (integer division!) — use 5.0 / 2 for 2.5.
  • int overflow: max ~2.1 billion; use long long for large numbers.
  • Mixing signed and unsigned comparisons causes subtle bugs.
  • Not initializing local variables (garbage values).

🌟 Interview Tips

  • Always use long long for competitive programming unless you're sure values fit in int.
  • Prefer static_cast<T>() over C-style casts: it's explicit and searchable.
  • Mention auto as a modern type-inference tool — shows C++11 knowledge.

📚 Simple Explanation

Python is dynamically typed: types are attached to objects, not variables. A variable is just a name that can point to any object.

  • Primitives (immutable): int, float, bool, str, tuple
  • Non-primitives (mutable): list, dict, set
  • Python integers have arbitrary precision (no overflow!).

📄 Syntax Snapshot

x = 10          # int
y = 3.14        # float
s = "hello"     # str
b = True        # bool
n = None        # None type

# Type check
type(x)         # <class 'int'>
isinstance(x, int)  # True

# Conversion
int("42")       # 42
str(99)         # "99"
float(3)        # 3.0

▶ Example Code

x = 10
x = "ten"       # reassigning with different type is OK

age  = int(input("Enter age: "))
gpa  = float("9.1")
name = str(42)  # "42"

# Python int has no overflow
big = 10 ** 100  # works fine
print(big)

💡 Real-World Analogy

Variables are like sticky labels on boxes. You can peel the label off one box and stick it on another (reassign to different type). The box (object) stays until nobody's pointing at it (garbage collected).

⚠ Common Mistakes

  • input() always returns a string — wrap with int()/float().
  • Assigning one list to another creates a reference, not a copy: b = a.
  • Shadowing built-in names: list = [1, 2] breaks the list built-in.
  • True and False are capitalized in Python.

🌟 Interview Tips

  • Explain: "Python passes object references by value — mutation affects original, reassignment does not."
  • Use type hints in interview code: def add(a: int, b: int) -> int: for clarity.
  • Python's int handles arbitrarily large numbers — great for factorial/big-number problems.

📚 Simple Explanation

JavaScript is dynamically typed with 7 primitive types and objects. Variables declared with const (can't reassign), let (can reassign, block-scoped), or var (avoid — function-scoped, hoisted).

  • Primitives: number, string, boolean, undefined, null, symbol, bigint
  • Non-primitives: objects, arrays, functions
  • All numbers (integers + floats) are IEEE 754 64-bit floating point.

📄 Syntax Snapshot

const pi   = 3.14;    // cannot reassign
let   age  = 20;      // can reassign
var   old  = "avoid"; // function-scoped, avoid

// Type check
typeof 42        // "number"
typeof "hi"      // "string"
typeof null      // "object" (famous JS bug)

// Conversion
Number("42")     // 42
String(99)       // "99"
Boolean(0)       // false

▶ Example Code

const name = "Alice";
let   score = 95;

score += 5;           // 100

const obj = { x: 1, y: 2 };  // non-primitive
const arr = [1, 2, 3];        // non-primitive

console.log(typeof name);  // "string"
console.log(typeof arr);   // "object"

💡 Real-World Analogy

const is a sealed box (reference can't change), let is an open shelf (you can swap items), var is a messy junk drawer shared across the whole room.

⚠ Common Mistakes

  • typeof null === "object" — a historical JS bug, not a feature.
  • 0.1 + 0.2 !== 0.3 — floating-point precision; use toFixed() for display.
  • Confusing undefined (declared but not assigned) vs null (intentionally empty).
  • Using var — causes hoisting confusion and function scope issues.

🌟 Interview Tips

  • Always use const first, downgrade to let only when mutation is needed.
  • List the 7 primitives confidently — it's a classic JS basics question.
  • Explain: "Objects and arrays are passed by reference value — mutation inside a function affects the original."
