Standard Libraries

The most useful built-in containers, algorithms, and utility functions — know these cold before interviews.

📚 Simple Explanation

  • STL (Standard Template Library): containers + algorithms + iterators.
  • Key containers: vector, map, unordered_map, set, queue, stack, priority_queue.
  • Key algorithms: sort, lower_bound, upper_bound, max/min, accumulate.

📄 Syntax Snapshot

#include <map>
#include <unordered_map>
#include <set>
#include <queue>
#include <algorithm>
#include <numeric>

std::map<string,int> freq;          // ordered O(log N)
std::unordered_map<string,int> uf;  // hash O(1) avg
std::set<int> s;                    // unique sorted
std::priority_queue<int> maxHeap;   // max-heap (default)
std::priority_queue<int, vector<int>, greater<int>> minHeap;

int sum = std::accumulate(v.begin(), v.end(), 0);
auto it = std::lower_bound(v.begin(), v.end(), target);

▶ Example Code

#include <iostream>
#include <unordered_map>
#include <vector>
#include <algorithm>
using namespace std;

// Frequency count with unordered_map
vector<int> topKFrequent(vector<int>& nums, int k) {
    unordered_map<int, int> freq;
    for (int n : nums) freq[n]++;

    vector<pair<int,int>> entries(freq.begin(), freq.end());
    sort(entries.begin(), entries.end(),
         [](auto& a, auto& b){ return a.second > b.second; });

    vector<int> result;
    for (int i = 0; i < k; i++) result.push_back(entries[i].first);
    return result;
}

💡 Real-World Analogy

The STL is like a fully stocked professional kitchen: you don't need to build your pots, knives, or measuring cups from scratch — professionals just know which tool to grab and use it.

⚠ Common Mistakes

  • Using map when unordered_map suffices — unnecessary O(log N) overhead.
  • Accessing a missing key in map with [] inserts a default value — use .count() or .find() first.
  • priority_queue is a max-heap by default — invert values or use greater<> for min-heap.

🌟 Interview Tips

  • Memorize: unordered_map = O(1) avg, worst O(N); map = O(log N) guaranteed.
  • Know lower_bound/upper_bound for binary search on sorted arrays.
  • priority_queue is the go-to for "top K" and median-finding problems.

📚 Simple Explanation

  • collections: Counter, defaultdict, deque, OrderedDict.
  • heapq: min-heap (negate values for max-heap).
  • itertools: permutations, combinations, product, groupby.
  • bisect: binary search on sorted lists.

📄 Syntax Snapshot

from collections import Counter, defaultdict, deque
import heapq
from bisect import bisect_left
import itertools

c = Counter("aabbcc")   # {'a':2,'b':2,'c':2}
c.most_common(2)         # top 2

d = defaultdict(int)     # no KeyError on missing key
d["x"] += 1

q = deque([1,2,3])
q.appendleft(0); q.popleft()  # O(1) both ends

heap = [3,1,2]
heapq.heapify(heap)         # min-heap in-place
heapq.heappush(heap, 0)
heapq.heappop(heap)         # pops smallest

▶ Example Code

from collections import Counter
import heapq

def top_k_frequent(nums, k):
    count = Counter(nums)           # {num: freq}
    # Use heap to get top k
    return heapq.nlargest(k, count.keys(), key=count.get)

print(top_k_frequent([1,1,1,2,2,3], 2))  # [1, 2]

# bisect for binary search
from bisect import bisect_left
arr = [1, 3, 5, 7, 9]
pos = bisect_left(arr, 5)  # 2 — index where 5 would be inserted

💡 Real-World Analogy

Python's standard library is like a Swiss Army knife with clear labels. Counter counts things, deque stacks/queues efficiently, heapq keeps the smallest item on top — each tool has one clear job.

⚠ Common Mistakes

  • Python's heapq is a min-heap: to simulate max-heap, insert -value.
  • Using a regular dict where defaultdict would save a get(key, default) call.
  • Calling list.sort() instead of using sorted() when you need the original list preserved.

🌟 Interview Tips

  • Counter(arr).most_common(k) solves frequency-top-K in one line.
  • Know itertools.combinations(arr, r) and permutations(arr, r) for combinatorics problems.
  • bisect_left/bisect_right is O(log N) search on a sorted list.

📚 Simple Explanation

  • Array methods: map, filter, reduce, find, some, every, flat, flatMap.
  • Map / Set: built-in hash map and unique-value set with O(1) operations.
  • Math: Math.max/min/floor/ceil/round/sqrt/pow/abs/log.
  • JSON: JSON.stringify / JSON.parse for serialization.

📄 Syntax Snapshot

const arr = [1, 2, 3, 4, 5];
arr.map(x => x * 2);           // [2,4,6,8,10]
arr.filter(x => x % 2 === 0);  // [2,4]
arr.reduce((acc, x) => acc + x, 0); // 15
arr.find(x => x > 3);          // 4
arr.some(x => x > 4);          // true
arr.every(x => x > 0);         // true

// Map & Set
const map = new Map();
map.set("a", 1); map.get("a");  // 1

const set = new Set([1,2,2,3]); // {1,2,3}
set.has(2); set.size;           // true, 3

▶ Example Code

// Frequency count with Map
function topKFrequent(nums, k) {
    const freq = new Map();
    for (const n of nums) freq.set(n, (freq.get(n) || 0) + 1);

    return [...freq.entries()]
        .sort((a, b) => b[1] - a[1])
        .slice(0, k)
        .map(e => e[0]);
}

console.log(topKFrequent([1,1,1,2,2,3], 2)); // [1, 2]

// Remove duplicates with Set
const unique = [...new Set([1,2,2,3,3,4])]; // [1,2,3,4]

💡 Real-World Analogy

JS array methods are like an assembly line: map transforms each item, filter screens defects out, reduce combines everything into a final product. Chain them together for elegant data pipelines.

⚠ Common Mistakes

  • Using a plain {} object as a hash map — watch out for prototype keys like "constructor"; prefer new Map().
  • reduce without an initial value crashes on an empty array.
  • Forgetting that Set.size is a property, not a method (unlike Java's .size()).

🌟 Interview Tips

  • new Map() maintains insertion order and allows any key type — use it over plain objects for hash maps.
  • Array deduplication: [...new Set(arr)] — clean one-liner interviewers love.
  • Know Array.from({length: n}, (_, i) => i) to create range arrays.
