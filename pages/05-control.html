Control Flow

Conditionals, loops, break/continue — the skeleton of every algorithm.

📚 Simple Explanation

  • if / else if / else — classic conditional branching.
  • switch — multi-way branch on integer/enum/char.
  • for — classic 3-part loop (init; cond; update).
  • range-based for — iterate containers cleanly (C++11).
  • while — loop while condition is true.
  • do-while — execute at least once, then check condition.
  • break — exit loop/switch immediately.
  • continue — skip to next iteration.

📄 Syntax Snapshot

if (x > 0) { ... } else if (x == 0) { ... } else { ... }

switch (x) { case 1: ...; break; default: ...; break; }

for (int i = 0; i < n; i++) { ... }
for (const auto& item : container) { ... }
while (condition) { ... }
do { ... } while (condition);

▶ Example Code

#include <iostream>
#include <vector>
int main() {
    std::vector<int> v = {1, 2, 3, 4, 5};

    for (const auto& x : v) {
        if (x == 3) continue;  // skip 3
        if (x == 5) break;     // stop at 5
        std::cout << x << " ";
    }
    // Output: 1 2 4

    // do-while: runs at least once
    int n = 0;
    do {
        n++;
    } while (n < 3);
    return 0;
}

💡 Real-World Analogy

Control flow is like a road network: if/else is a turn, switch is a roundabout with exits, for/while is driving in circles until you meet the exit condition.

⚠ Common Mistakes

  • Forgetting break in switch — causes fall-through to next case.
  • Off-by-one: i < n vs i <= n — draw it out when in doubt.
  • Infinite loop: forgetting to update the loop variable.
  • Modifying a container while range-looping over it.

🌟 Interview Tips

  • Always state loop bounds clearly: "i goes from 0 inclusive to n exclusive."
  • Prefer range-based for loop for readability — it also avoids index out-of-bounds.
  • If a problem has nested loops, proactively mention the O(N²) complexity.

📚 Simple Explanation

  • if / elif / else — conditional branching (no switch before 3.10).
  • match / case — pattern matching (Python 3.10+).
  • for x in iterable — iterates values directly (no index needed).
  • while — loop with condition.
  • No do-while — simulate with while True: ... if cond: break.
  • for-else / while-else — unique: else runs only if no break.

📄 Syntax Snapshot

if x > 0:
    pass
elif x == 0:
    pass
else:
    pass

for i in range(n):         # 0..n-1
for i, v in enumerate(arr): # index + value
while condition:
    pass

for x in arr:
    ...
else:
    # runs if no break was hit
    ...

▶ Example Code

arr = [1, 2, 3, 4, 5]

for x in arr:
    if x == 3:
        continue   # skip 3
    if x == 5:
        break      # stop at 5
    print(x, end=" ")
# Output: 1 2 4

# for-else: useful for search
for x in arr:
    if x == 99:
        print("found")
        break
else:
    print("not found")   # prints this

💡 Real-World Analogy

for x in list is like picking items one-by-one from a conveyor belt — you don't manage the belt position, you just grab each item. The else block fires when the belt runs out naturally (no break).

⚠ Common Mistakes

  • range(n) is 0 to n-1 (exclusive end) — visualize before coding.
  • Modifying a list while iterating: iterate over a copy (for x in arr[:]).
  • Forgetting that Python has no do-while.
  • Misusing for-else: the else runs when no break occurred — easy to misread.

🌟 Interview Tips

  • Use enumerate instead of manual index counters.
  • Use zip(a, b) to iterate two sequences in parallel.
  • Mention for-else when implementing search — it impresses interviewers with Python knowledge depth.

📚 Simple Explanation

  • if / else if / else — standard branching.
  • switch — multi-way branch (with mandatory break).
  • for — classic C-style.
  • for...of — iterate values of an iterable (arrays, strings).
  • for...in — iterate keys of an object (avoid on arrays!).
  • while / do-while — condition-based loops.
  • Array methods — forEach, map, filter, reduce.

📄 Syntax Snapshot

for (let i = 0; i < n; i++) { ... }
for (const val of arr)  { ... }  // values
for (const key in obj)  { ... }  // keys

while (condition) { ... }
do { ... } while (condition);

arr.forEach(x => console.log(x));
arr.map(x => x * 2).filter(x => x > 4);

▶ Example Code

const arr = [1, 2, 3, 4, 5];

// for...of
for (const x of arr) {
  if (x === 3) continue;
  if (x === 5) break;
  console.log(x);  // 1 2 4
}

// Functional style
const evens  = arr.filter(x => x % 2 === 0);  // [2, 4]
const doubled = evens.map(x => x * 2);          // [4, 8]
const sum    = arr.reduce((acc, x) => acc + x, 0); // 15

💡 Real-World Analogy

for...of picks items from a queue. for...in reads labels on drawers (object keys). Using for...in on an array reads "0", "1", "2"… (the indexes as strings) — usually not what you want.

⚠ Common Mistakes

  • Using for...in on an array — you get string indices, not values.
  • Forgetting break in switch — fall-through is silent and dangerous.
  • forEach can't be stopped with break — use a for...of loop when you need early exit.
  • Mutation inside map/filter — these should be pure (no side effects).

🌟 Interview Tips

  • Prefer for...of over classic for for arrays — cleaner and error-free.
  • Know map/filter/reduce well — they're extremely common in frontend interviews.
  • When you need break, use for...of; when you want transformation, use map/filter.
