Operators

Arithmetic, relational, logical, bitwise, and ternary operators — plus their gotchas.

📚 Simple Explanation

  • Arithmetic: + - * / % — / is integer division if both operands are integers.
  • Relational: == != < > <= >= — returns bool.
  • Logical: && (AND), || (OR), ! (NOT) — short-circuit evaluation.
  • Bitwise: & | ^ ~ << >> — operates on individual bits.
  • Assignment: = += -= *= /= %=
  • Ternary: cond ? a : b

📄 Syntax Snapshot

a / b           // integer division if both int
a % b           // remainder
a && b          // logical AND
a & b           // bitwise AND (different!)
(n & 1) == 0    // check if even
cond ? x : y    // ternary

▶ Example Code

#include <iostream>
int main() {
    int a = 7, b = 3;

    std::cout << a / b  << "\n"; // 2 (int division)
    std::cout << a % b  << "\n"; // 1
    std::cout << (a > b) << "\n"; // 1 (true)

    // Bitwise: check even/odd
    std::cout << ((a & 1) ? "odd" : "even") << "\n";

    // Left shift = multiply by 2
    std::cout << (a << 1) << "\n"; // 14

    return 0;
}

💡 Real-World Analogy

Operators are like different tools in a toolbox: a hammer (arithmetic), ruler (relational), logical gates (logical), lamp toggle (bitwise). Each does one job — mixing them up causes errors.

⚠ Common Mistakes

  • 5 / 2 == 2 not 2.5 — cast to double first if you need fractions.
  • Confusing && (logical) with & (bitwise).
  • Using = instead of == in a condition: if (x = 5) always true!
  • Operator precedence surprises without parentheses.

🌟 Interview Tips

  • Know bitwise tricks: n & (n-1) clears lowest set bit, useful for power-of-two checks.
  • Short-circuit: ptr != nullptr && ptr->val == x is safe because of left-to-right evaluation.
  • Ternary is fine for simple inline conditionals; avoid nesting it.

📚 Simple Explanation

  • Arithmetic: + - * / (true division) // (floor div) % ** (power)
  • Relational: == != < > <= >= — chainable: 1 < x < 10
  • Logical: and, or, not — use words, not symbols
  • Bitwise: & | ^ ~ << >>
  • Identity: is, is not — compare object identity
  • Membership: in, not in
  • Ternary: a if condition else b

📄 Syntax Snapshot

7 / 2     # 3.5  (true division!)
7 // 2    # 3    (floor division)
7 % 2     # 1
2 ** 10   # 1024 (power)

x = 5
result = "big" if x > 3 else "small"

1 < x < 10  # chained comparison (unique to Python!)

▶ Example Code

a, b = 7, 3

print(a // b)   # 2  (floor division)
print(a / b)    # 2.333...  (true division)
print(a ** b)   # 343

# Logical operators
print(a > 5 and b < 5)  # True

# Membership
arr = [1, 2, 3]
print(2 in arr)           # True

# Ternary
label = "odd" if a % 2 else "even"
print(label)

💡 Real-World Analogy

Python logical operators read like everyday English: "if it rains and I have an umbrella, go out." Much more natural than && and ||.

⚠ Common Mistakes

  • Forgetting that / is always true division in Python 3 — use // for int result.
  • Using & instead of and for boolean logic (bitwise vs logical).
  • Using == for identity checks: use is for singletons like None.
  • not in and is not are two-word operators — don't break them up.

🌟 Interview Tips

  • Chained comparisons (0 <= x < n) are Pythonic and impressive.
  • Use x is None, not x == None.
  • Walrus operator := (Python 3.8+): if (n := len(a)) > 10: — assign inside expression.

📚 Simple Explanation

  • Arithmetic: + - * / % ** — + also concatenates strings (careful!)
  • Relational: < > <= >=, strict ===, loose ==
  • Logical: &&, ||, !, nullish coalescing ??
  • Bitwise: & | ^ ~ << >> >>>
  • Ternary: cond ? a : b
  • Optional chaining: obj?.prop (safe access)

📄 Syntax Snapshot

5 == "5"    // true  (type coercion)
5 === "5"   // false (strict — prefer this)

null ?? "default"  // "default" (nullish coalescing)
obj?.name          // undefined (not error) if obj is null
x ||= 3            // x = x || 3 (logical OR assignment)

▶ Example Code

const a = 7, b = 3;

console.log(a / b);          // 2.333...(float!)
console.log(Math.floor(a/b)); // 2 (manual floor div)
console.log(a ** b);         // 343

// ALWAYS use ===
console.log(a === 7);  // true
console.log(a == "7"); // true (dangerous coercion!)

// Ternary
const label = (a % 2 === 0) ? "even" : "odd";
console.log(label); // odd

// Nullish coalescing
const name = null;
console.log(name ?? "Guest"); // "Guest"

💡 Real-World Analogy

== is like asking "Do you look similar?" — a number and a string might match. === is like asking "Are you the exact same person (type + value)?" — much safer.

⚠ Common Mistakes

  • Using == — always use === to avoid coercion surprises.
  • "5" - 3 = 2 but "5" + 3 = "53" — subtraction coerces, addition concatenates.
  • Confusing || (falsy fallback) with ?? (null/undefined fallback) — 0 || "x" = "x" but 0 ?? "x" = 0.

🌟 Interview Tips

  • Lead with === — every JS interviewer notices if you use ==.
  • Know ?? and ?. — ES2020 features that show modern JS awareness.
  • Demonstrate short-circuit: user && user.name is a common pattern in real code.
