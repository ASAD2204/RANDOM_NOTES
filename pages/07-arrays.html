Arrays & Strings

The most common data structures in interview problems — slicing, traversal, mutation, and complexity.

📚 Simple Explanation

  • C-array: fixed-size, on stack/heap. Rarely used directly.
  • std::vector: dynamic array, preferred in interviews.
  • std::string: mutable character sequence — supports append, substr, find.
  • Indexing is O(1). push_back is amortized O(1). Insert in middle is O(N).

📄 Syntax Snapshot

#include <vector>
#include <string>

std::vector<int> v = {1, 2, 3};
v.push_back(4);          // append
v.size();                // length
v[0];                    // access (no bounds check!)
v.at(0);                 // access (throws if out of range)

std::string s = "hello";
s += "!";                // append
s.substr(1, 3);          // "ell"
s.length();              // 6

▶ Example Code

#include <iostream>
#include <vector>
#include <string>
#include <algorithm>

int main() {
    std::vector<int> v = {5, 3, 1, 4, 2};
    std::sort(v.begin(), v.end());  // [1,2,3,4,5]

    // Two-pointer reverse string
    std::string s = "abcde";
    int l = 0, r = s.size() - 1;
    while (l < r) std::swap(s[l++], s[r--]);
    std::cout << s << "\n"; // "edcba"

    return 0;
}

💡 Real-World Analogy

A vector is like a resizable shelf: fast lookup by slot number (O(1)), cheap to add at the end, but slow to insert in the middle (everything shifts).

⚠ Common Mistakes

  • Out-of-bounds access with [] — silent UB; use .at() for safety during debugging.
  • Mixing size_t (unsigned) and int in comparisons — negative int becomes huge unsigned.
  • String concatenation in a loop with += for very large strings — prefer std::stringstream or pre-allocation.
  • Forgetting that substr(pos, length) second arg is length, not end index.

🌟 Interview Tips

  • Know complexities: index O(1), push_back amortized O(1), insert/erase O(N).
  • For 2D grids: vector<vector<int>> is standard.
  • Two-pointer technique on arrays and strings solves many O(N) problems neatly.

📚 Simple Explanation

  • list: mutable dynamic array; any type can mix.
  • str: immutable sequence of characters — every operation creates a new string.
  • Slicing: s[start:stop:step] — extremely powerful.
  • List comprehension: concise alternative to loops.

📄 Syntax Snapshot

arr = [1, 2, 3]
arr.append(4)           # O(1)
arr.insert(0, 0)        # O(N) — shifts
arr.pop()               # O(1) remove last
arr.pop(0)              # O(N) remove first

s = "hello"
s[::-1]                 # "olleh" (reverse)
s[1:4]                  # "ell"
s.split(",")            # list of substrings
",".join(["a","b","c"]) # "a,b,c"

▶ Example Code

arr = [5, 3, 1, 4, 2]
arr.sort()          # [1,2,3,4,5] in-place
sorted_copy = sorted(arr)  # returns new list

# Two-pointer reverse
s = list("abcde")
l, r = 0, len(s) - 1
while l < r:
    s[l], s[r] = s[r], s[l]
    l += 1; r -= 1
print("".join(s))   # "edcba"

# List comprehension
squares = [x**2 for x in range(5)]  # [0,1,4,9,16]

💡 Real-World Analogy

A Python list is a shopping trolley: throw anything in, pick by position, rearrange as you like. A string is text engraved on stone: read it freely, but you can't chip changes in — you must carve a new stone.

⚠ Common Mistakes

  • Trying to mutate a string: s[0] = 'H' raises TypeError.
  • Concatenating strings in a loop: result += s is O(N²); use "".join(list).
  • arr.sort() returns None — don't do arr = arr.sort().
  • Slicing end is exclusive: s[0:3] gives chars 0, 1, 2.

🌟 Interview Tips

  • Master slicing and list comprehensions — they're compact and impressive.
  • For frequency counts use Counter(arr) from collections.
  • Reverse a string in O(N): s[::-1] — mention it and know why it creates a new string.

📚 Simple Explanation

  • Array: dynamic, can hold mix of types, objects at heart.
  • String: immutable primitive; methods return new strings.
  • Rich built-in methods: push/pop, shift/unshift, splice, slice, indexOf, includes.

📄 Syntax Snapshot

const arr = [1, 2, 3];
arr.push(4);            // append O(1)
arr.unshift(0);         // prepend O(N)
arr.pop();              // remove last
arr.shift();            // remove first O(N)
arr.splice(1, 2);       // remove 2 elements at index 1

const s = "hello";
s.slice(1, 4);          // "ell"
s.split("");            // ['h','e','l','l','o']
arr.join("");           // "hello"
s.toUpperCase();        // new string

▶ Example Code

const arr = [5, 3, 1, 4, 2];
arr.sort((a, b) => a - b);  // [1,2,3,4,5] (numeric!)

// Reverse a string
const s = "abcde";
const rev = s.split("").reverse().join("");
console.log(rev); // "edcba"

// Two-pointer in-place
const chars = s.split("");
let l = 0, r = chars.length - 1;
while (l < r) {
  [chars[l], chars[r]] = [chars[r], chars[l]];
  l++; r--;
}
console.log(chars.join("")); // "edcba"

💡 Real-World Analogy

A JS array is like a numbered locker row that can expand. A string is an engraved name plate — you can read every letter but only get a new plate (new string) to change it.

⚠ Common Mistakes

  • arr.sort() without a comparator sorts lexicographically: [10,9,2].sort() = [10,2,9]!
  • Using delete arr[i] creates an empty slot (hole), not removal — use splice.
  • Mutating an array passed to a function unexpectedly — pass a copy: [...arr].

🌟 Interview Tips

  • Always pass a comparator to .sort() for numbers: (a, b) => a - b.
  • Know map/filter/reduce — used in nearly every frontend interview.
  • Spread operator [...arr] is the easiest shallow clone — mention it.
