Functions

Defining reusable logic — parameters, return values, recursion, lambdas, and closures.

📚 Simple Explanation

Functions in C++ have an explicit return type and can accept parameters by value, reference (&), or pointer (*). Function overloading allows same name with different signatures.

  • Pass by value — copy of argument, original unchanged.
  • Pass by reference — alias to original, can modify.
  • Pass by const reference — read-only alias (efficient for large objects).
  • Recursion — function calls itself; needs a base case.
  • Lambda — anonymous function (C++11): [capture](params){ ... }

📄 Syntax Snapshot

returnType name(paramType param) { return value; }

void     swap(int& a, int& b) { ... }       // by reference
int      sum(const std::vector<int>& v) { } // const ref

// Lambda
auto greet = [](const std::string& s) {
    return "Hello " + s;
};

▶ Example Code

#include <iostream>

int factorial(int n) {          // recursion
    if (n <= 1) return 1;       // base case
    return n * factorial(n - 1);
}

void swap(int& a, int& b) {    // pass by reference
    int tmp = a; a = b; b = tmp;
}

int main() {
    std::cout << factorial(5) << "\n"; // 120

    int x = 3, y = 7;
    swap(x, y);
    std::cout << x << " " << y << "\n"; // 7 3
    return 0;
}

💡 Real-World Analogy

A function is a vending machine: you put in inputs (coins), it processes internally, and returns an output (snack). Recursion is calling a smaller vending machine inside the big one (Russian dolls).

⚠ Common Mistakes

  • Missing recursion base case → infinite recursion → stack overflow.
  • Returning a reference to a local variable (dangling reference).
  • Passing large objects by value (copies them) — use const& instead.
  • Function overloads being ambiguous (compiler can't resolve which to call).

🌟 Interview Tips

  • Always name functions clearly: isPrime, findMax — self-documenting code.
  • When using recursion, state: base case, recursive case, and stack depth (O(N) space).
  • Prefer const& for read-only parameters over plain value for non-trivial types.

📚 Simple Explanation

Python functions are defined with def. They're first-class objects — assigned to variables, passed as arguments, returned. Key features:

  • Default arguments — def f(x, y=0)
  • *args — variable positional arguments (tuple)
  • **kwargs — variable keyword arguments (dict)
  • Lambda — lambda x: x*2
  • Closures — inner function captures outer variable.

📄 Syntax Snapshot

def function_name(params, default=value):
    return result

def variadic(*args, **kwargs):
    pass

# Lambda
square = lambda x: x ** 2

# Type hints (good practice)
def add(a: int, b: int) -> int:
    return a + b

▶ Example Code

def factorial(n: int) -> int:
    if n <= 1:
        return 1          # base case
    return n * factorial(n - 1)

def greet(name: str, greeting: str = "Hello") -> str:
    return f"{greeting}, {name}!"

# Lambda in sorting
people = [("Alice", 30), ("Bob", 25)]
people.sort(key=lambda p: p[1])  # sort by age

print(factorial(5))        # 120
print(greet("Alice"))      # Hello, Alice!

💡 Real-World Analogy

A function is a recipe card: same instructions, different ingredients (arguments). Default arguments are ingredient substitutes listed on the card. A closure is a chef who remembers the secret sauce from the parent kitchen.

⚠ Common Mistakes

  • Mutable default argument: def f(x=[]) — the list is shared across all calls! Use x=None.
  • Forgetting return — function returns None silently.
  • Shadowing built-ins with parameter names: def f(list, ...).
  • Recursion hitting Python's default limit (~1000). Use iterative or sys.setrecursionlimit.

🌟 Interview Tips

  • Add type hints — interviewers appreciate them as documentation.
  • Python passes object references by value: mutation inside affects original; reassignment does not.
  • Show closure knowledge: "Inner functions can read outer scope variables. To write, use nonlocal."

📚 Simple Explanation

Functions are first-class citizens: stored, passed, returned. Three main ways to define:

  • Function declaration — hoisted, can be called before definition.
  • Function expression — assigned to a variable, not hoisted.
  • Arrow function — concise syntax, this inherited from parent scope.
  • Closures — function remembers variables from its birthplace scope.
  • IIFE — Immediately Invoked Function Expression.

📄 Syntax Snapshot

// Declaration (hoisted)
function greet(name) { return `Hi ${name}`; }

// Expression (not hoisted)
const greet = function(name) { return `Hi ${name}`; };

// Arrow (concise + lexical this)
const greet = name => `Hi ${name}`;
const add   = (a, b) => a + b;

// Default params
const multiply = (a, b = 1) => a * b;

// Rest params
const sum = (...nums) => nums.reduce((t, n) => t + n, 0);

▶ Example Code

function factorial(n) {
  if (n <= 1) return 1;
  return n * factorial(n - 1);
}

// Closure example
function makeCounter() {
  let count = 0;
  return () => ++count;  // arrow captures 'count'
}
const counter = makeCounter();
console.log(counter()); // 1
console.log(counter()); // 2
console.log(counter()); // 3

console.log(factorial(5)); // 120

💡 Real-World Analogy

A closure is like a backpack you take from a camping trip: even after you leave the campsite (outer function ends), the supplies (variables) in your backpack (closure) are still there for use.

⚠ Common Mistakes

  • Using this inside an arrow function expecting it to refer to the object — arrow functions inherit this from the outer context.
  • Relying on hoisting for function expressions — only declarations are hoisted.
  • Creating closures in a loop with var: all closures share the same final i; use let instead.

🌟 Interview Tips

  • Closures are one of the most common JS interview topics — be ready to explain and demonstrate with a counter example.
  • Arrow vs regular function for this differences — classic in object/class questions.
  • Mention higher-order functions (functions that take/return functions) — shows functional programming awareness.
