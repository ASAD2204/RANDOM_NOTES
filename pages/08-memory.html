Memory Concepts

Stack vs heap, pointers, smart pointers, garbage collection, and shallow vs deep copy.

📚 Simple Explanation

  • Stack: auto-managed; fast; local variables live here; freed when scope ends.
  • Heap: manual; dynamic allocation with new; must free with delete.
  • Pointer: holds a memory address. *ptr = dereference, &x = address-of.
  • Smart pointers (unique_ptr, shared_ptr) automate deletion — no memory leaks.

📄 Syntax Snapshot

int x = 5;           // stack
int* p = new int(5); // heap
*p = 10;             // dereference
delete p;            // free heap
p = nullptr;         // avoid dangling pointer

// Smart pointers (prefer these!)
#include <memory>
auto up = std::make_unique<int>(42);    // unique owner
auto sp = std::make_shared<int>(42);    // shared ownership

▶ Example Code

#include <iostream>
#include <memory>

struct Node {
    int val;
    Node* next = nullptr;
};

int main() {
    // Proper memory management with smart pointers
    auto head = std::make_unique<Node>();
    head->val = 1;
    // head auto-deleted when out of scope — no memory leak

    // Shallow vs deep copy (raw arrays)
    int a[] = {1, 2, 3};
    int* b = a; // shallow — both point to same data
    b[0] = 99;
    std::cout << a[0]; // 99 — a changed too!

    return 0;
}

💡 Real-World Analogy

Stack is a whiteboard: fast, erased automatically after a meeting. Heap is a storage unit: you rent it (new), things stay as long as you want, but you must remember to cancel the lease (delete) or you pay forever (memory leak).

⚠ Common Mistakes

  • Memory leak: Calling new without matching delete.
  • Dangling pointer: Accessing pointer after delete; always set to nullptr.
  • Double free: Calling delete twice on the same pointer.
  • Stack overflow: Deeply recursive function or very large local arrays.

🌟 Interview Tips

  • Always prefer make_unique/make_shared over raw new in modern C++.
  • Be able to explain the difference between shallow and deep copy.
  • Interviewers love asking about RAII — Resource Acquisition Is Initialization.

📚 Simple Explanation

  • Python uses automatic garbage collection — reference counting + cycle collector.
  • Variables are references (labels) pointing to objects in memory.
  • Shallow copy: new container, same inner objects.
  • Deep copy: new container and recursive copies of all inner objects.

📄 Syntax Snapshot

import copy

# Assignment vs copy
a = [1, [2, 3]]
b = a           # same reference
c = a.copy()    # shallow copy
d = copy.deepcopy(a)  # deep copy

b.append(4)     # a also changes!
c[1].append(9)  # a[1] also changes (shallow — inner list shared)
d[1].append(9)  # a unchanged (deep copy)

# Reference counting
import sys
x = [1, 2, 3]
sys.getrefcount(x)  # usually 2 (x + getrefcount arg)

▶ Example Code

import copy

original = [[1, 2], [3, 4]]

shallow = original.copy()     # or list(original)
deep    = copy.deepcopy(original)

original[0].append(99)

print(shallow[0])  # [1, 2, 99] — inner list is shared
print(deep[0])     # [1, 2]     — completely independent

# id() shows memory address
x = 5
y = 5
print(x is y)  # True — Python interns small ints

💡 Real-World Analogy

A variable is a sticky note with a room number written on it — the object is in the room. Assignment copies the note (same room). Shallow copy makes a new folder containing the original room numbers. Deep copy rebuilds every room from scratch.

⚠ Common Mistakes

  • Assuming b = a copies a list — it just copies the reference.
  • Using a.copy() on a nested list and expecting full independence.
  • Circular references can delay garbage collection (use gc module to diagnose).

🌟 Interview Tips

  • Know the difference between is (identity) and == (equality).
  • Small integer caching: -5 to 256 are pre-allocated singletons.
  • Be ready to explain Python's garbage collection: ref-counting + cyclic GC.

📚 Simple Explanation

  • JS uses an automatic garbage collector (mark-and-sweep in V8).
  • Primitives are copied by value; objects/arrays are copied by reference.
  • Shallow copy: Object.assign or spread {...obj}.
  • Deep copy: structuredClone(obj) (modern) or JSON.parse(JSON.stringify(obj)) (legacy).

📄 Syntax Snapshot

// Primitives — copied by value
let a = 5;
let b = a;
b = 10;  // a is still 5

// Objects — copied by reference
const obj = { x: 1 };
const ref = obj;
ref.x = 99;  // obj.x is also 99!

// Shallow copy
const shallow = { ...obj };
const shallow2 = Object.assign({}, obj);

// Deep copy
const deep = structuredClone(obj);   // modern, handles nested
const deep2 = JSON.parse(JSON.stringify(obj)); // no functions/undefined

▶ Example Code

const original = { name: "Alice", scores: [90, 80] };

const shallow = { ...original };
const deep = structuredClone(original);

original.scores.push(100);

console.log(shallow.scores); // [90, 80, 100] — shared inner array
console.log(deep.scores);    // [90, 80]       — independent

// Memory leak example (closure holding reference)
function leak() {
  const bigData = new Array(1000000).fill("x");
  return () => bigData.length; // bigData can't be GC'd
}

💡 Real-World Analogy

Primitives are like numbers written on paper — copying writes the same digits. Objects are like map coordinates: copying the coordinates still leads you to the same physical place. structuredClone builds a new identical place at separate coordinates.

⚠ Common Mistakes

  • Spreading nested objects and assuming full independence.
  • JSON.parse(JSON.stringify()) drops undefined, functions, Date objects, Map, Set.
  • Closures keeping large objects alive — a common JS memory leak pattern.

🌟 Interview Tips

  • Always clarify "copy" questions — shallow vs deep is a frequent follow-up.
  • Prefer structuredClone over JSON tricks when asked for modern approach.
  • The V8 engine's generational GC (young/old space) is good advanced knowledge.
