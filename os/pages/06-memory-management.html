Memory Management

Allocation, paging, segmentation, and fragmentation.

📚 Goals

  • Efficient memory utilization
  • Isolation and protection
  • Relocation and sharing support

Allocation strategies

First Fit
Best Fit
Worst Fit

Paging

Logical memory split into pages, physical memory into frames. Page table maps page -> frame.

Segmentation

Memory divided by logical units (code/data/stack), each with base + limit.

Fragmentation

Internal fragmentation: unused space inside allocated block.
External fragmentation: free memory split into small non-contiguous holes.

🌟 Interview Tips

  • Paging removes external fragmentation but may add internal fragmentation.
  • Segmentation matches programmer view but has external fragmentation risk.

▶ Theory Deep Dive

Memory management maps logical program space to physical memory while enforcing protection and efficient utilization. Fragmentation and allocation policy strongly affect performance.
