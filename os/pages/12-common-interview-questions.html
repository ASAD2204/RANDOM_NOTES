Common OS Interview Questions

Conceptual and scenario-based OS interview questions with answer direction.

▶ High-frequency conceptual questions

  1. Difference between process and thread?
  2. User mode vs kernel mode?
  3. What is context switch and why is it expensive?
  4. What is race condition and critical section?
  5. Mutex vs semaphore?
  6. Deadlock vs starvation?
  7. Paging vs segmentation?
  8. What is virtual memory and page fault?
  9. FIFO vs LRU page replacement differences?
  10. What are system calls?
  11. What is DMA?
  12. What is thrashing?

▶ Scenario-based questions

  1. A web server becomes slow with high context switching. What could be wrong?
  2. System shows frequent page faults and low CPU utilization. Diagnose.
  3. Two services deadlock over DB/file locks. How to recover and prevent?
  4. A producer-consumer queue occasionally corrupts data. Root cause?
  5. How would you choose RR time quantum for interactive workloads?

🌟 Strong answer pattern

Definition -> Cause -> Example -> Trade-off -> Mitigation

▶ Theory Deep Dive

Strong OS interview answers link mechanism to trade-off: correctness, performance, fairness, and failure handling in concurrent system behavior.
