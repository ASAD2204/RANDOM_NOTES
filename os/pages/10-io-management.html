I/O Management

Device management, buffering, caching, and spooling basics.

📚 Concept

OS manages device controllers and drivers, coordinating data transfer between CPU, memory, and peripherals.

I/O techniques

Programmed I/O
Interrupt-driven I/O
DMA (Direct Memory Access)

Buffering techniques

Single buffering
Double buffering
Circular buffering

Caching vs Buffering vs Spooling

  • Buffering: smooth producer-consumer speed mismatch.
  • Caching: keep frequently used data for faster access.
  • Spooling: queue I/O jobs (e.g., print spooler).

🌟 Interview Tips

  • DMA reduces CPU overhead for bulk data transfer.
  • Interrupt-driven I/O is more efficient than busy waiting.

▶ Theory Deep Dive

I/O management coordinates slow, asynchronous devices with CPU execution. Interrupts, DMA, buffering, and scheduling reduce idle CPU time and improve throughput.
