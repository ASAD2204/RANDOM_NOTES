Processes and Threads

Lifecycle, PCB, process states, and process vs thread differences.

📚 Process

A process is a program in execution with its own address space, resources, and execution context.

📚 Thread

A thread is the smallest execution unit within a process. Threads share process memory/resources but have separate stacks/registers.

🔄 Process states

New -> Ready -> Running -> Waiting/Blocked -> Ready -> Terminated

📦 Process Control Block (PCB)

  • Process ID
  • Program counter
  • CPU registers
  • Scheduling info (priority, queue pointers)
  • Memory management info
  • I/O status/accounting info

▶ Process vs Thread

Process:
- Heavyweight
- Separate memory
- Costly context switch
- Strong isolation

Thread:
- Lightweight
- Shared memory in same process
- Faster context switch
- Synchronization needed for shared data

🌟 Interview Tips

  • Context switch between processes is costlier than thread switch.
  • Threads improve responsiveness but introduce race conditions.

▶ Theory Deep Dive

Processes isolate execution contexts; threads share process resources for concurrency. Correct system design balances isolation, communication overhead, and scheduling fairness.
