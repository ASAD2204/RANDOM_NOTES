Deadlocks

Conditions, prevention, avoidance, and detection strategies.

📚 What is deadlock?

A set of processes is deadlocked if each process waits for a resource held by another process in the set.

Necessary conditions (Coffman)

1) Mutual Exclusion
2) Hold and Wait
3) No Preemption
4) Circular Wait

Approaches

  • Prevention: break at least one Coffman condition.
  • Avoidance: allocate resources safely (e.g., Banker's algorithm).
  • Detection: allow deadlock, then detect and recover.

Detection idea

Use wait-for/resource allocation graph.
Cycle in wait-for graph => deadlock (single instance resource model).

🌟 Interview Tips

  • Differentiate deadlock vs starvation.
  • Mention practical recovery: kill process, rollback, preempt resource.

▶ Theory Deep Dive

Deadlocks emerge from circular wait with resource constraints. Prevention, avoidance, detection, and recovery represent different control strategies with different overhead profiles.
