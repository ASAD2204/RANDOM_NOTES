System Calls

How user programs request kernel services safely.

📚 Concept

System calls are controlled entry points from user mode to kernel mode for privileged operations.

Categories

Process control: fork, exec, wait, exit
File management: open, read, write, close
Device management: ioctl
Information maintenance: getpid, time
Communication: pipe, shared memory, socket

Call flow

User program -> library wrapper -> trap/syscall instruction -> kernel handler -> return to user mode

🌟 Interview Tips

  • Explain difference between API and system call.
  • Context switch and mode switch overhead are relevant in performance discussions.

▶ Theory Deep Dive

System calls are controlled transitions from user mode to kernel mode. They enforce privilege boundaries while exposing hardware/services through stable APIs.
