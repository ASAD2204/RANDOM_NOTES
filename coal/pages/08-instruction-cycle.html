Instruction Cycle

Fetch → Decode → Execute loop for every instruction.

▶ Steps

  1. Fetch: instruction loaded from memory using PC.
  2. Decode: CU interprets opcode and operands.
  3. Execute: ALU/memory/I/O operation performed.
  4. Write-back: result stored in register/memory (if needed).
  5. PC update: next instruction address selected.

▶ Simplified micro-operations

MAR <- PC
MDR <- M[MAR]
IR  <- MDR
Decode(IR)
Execute(IR)
PC <- PC + 1 (or branch target)
