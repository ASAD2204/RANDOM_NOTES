Input Output Organization

How CPU exchanges data with external devices efficiently.

▶ I/O transfer methods

  • Programmed I/O: CPU polls device status repeatedly.
  • Interrupt-driven I/O: device interrupts CPU when ready.
  • DMA: direct memory access controller transfers blocks with minimal CPU intervention.

▶ Addressing styles

  • Memory-mapped I/O: devices use normal memory address space.
  • Isolated I/O: separate I/O instruction space.

🌟 Trade-off summary

Polling: simple but wastes CPU
Interrupt: efficient for sporadic events
DMA: best for large block transfers

▶ Theory Deep Dive

Computer organization topics are best understood as abstraction layers from digital logic to CPU, memory, and I/O performance trade-offs.
