Pipelining

Overlaps instruction stages to improve throughput.

▶ 5-stage pipeline model

IF -> ID -> EX -> MEM -> WB
I1: IF ID EX MEM WB
I2:    IF ID EX MEM WB
I3:       IF ID EX MEM WB

Latency per instruction may not decrease, but instructions per unit time increase.

▶ Hazards

  • Structural: resource conflict.
  • Data: dependency between instructions (RAW most common).
  • Control: branch/jump uncertainty.

🌟 Hazard handling

  • Data forwarding/bypassing
  • Pipeline stalls (bubbles)
  • Branch prediction + speculative execution
