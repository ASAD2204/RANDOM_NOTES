Computer System Overview

How software instructions become electrical operations in hardware.

▶ Hardware and software interaction

Software is written in high-level languages, translated to machine instructions by compiler/assembler, loaded by OS into memory, and executed by CPU. Hardware executes instructions exactly; software decides what should be done.

Application -> OS -> ISA instructions -> CPU datapath/control -> Memory/I/O

▶ Main blocks and role

  • CPU: executes instructions.
  • Main Memory (RAM): stores active code/data.
  • Storage: persistent data (SSD/HDD).
  • I/O Devices: keyboard, display, network, disk controllers.
  • Buses: data/address/control communication channels.

🌟 Interview angle

Differentiate architecture (programmer-visible behavior: ISA) and organization (implementation details: pipelines, cache size, buses).

▶ Theory Deep Dive

Computer organization topics are best understood as abstraction layers from digital logic to CPU, memory, and I/O performance trade-offs.
