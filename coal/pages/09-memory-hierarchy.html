Memory Hierarchy

Balances speed, capacity, and cost using layered storage.

▶ Hierarchy levels

Fastest/Smallest/Costliest
Registers
L1/L2/L3 Cache
Main Memory (RAM)
Secondary Storage (SSD/HDD)
Slowest/Largest/Cheapest

▶ Why hierarchy works

Programs show temporal and spatial locality, so recently/frequently used data can stay in fast upper levels.

🌟 Interview phrase

Average Memory Access Time reduces by exploiting locality through multi-level caching.

▶ Theory Deep Dive

Computer organization topics are best understood as abstraction layers from digital logic to CPU, memory, and I/O performance trade-offs.
