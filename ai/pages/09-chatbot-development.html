Chatbot Development

From scripted bots to retrieval-augmented, tool-using conversational systems.

▶ Types of chatbots

  • Rule-based: deterministic flows, high control, low flexibility.
  • Retrieval-based: picks best answer from indexed knowledge.
  • Generative: synthesizes novel responses using language models.
  • LLM-based: combines prompts, memory, retrieval, and tools.

▶ Chatbot architecture

User -> API Gateway -> Orchestrator -> (Retriever / Tools / Memory) -> LLM
                                                                             -> Safety/Policy Layer -> Response

▶ Theory Deep Dive

A production chatbot is a policy-governed dialog system. The model generates language, but orchestration decides what context to include, which tools to call, and when to block unsafe outputs. High-quality systems optimize for correctness, consistency, response latency, and user trust rather than verbosity alone.

  • State management: maintain dialogue context across turns.
  • Grounding: attach factual context from internal knowledge sources.
  • Guardrails: enforce safety, domain policy, and escalation logic.
