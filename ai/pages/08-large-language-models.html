Large Language Models (LLMs)

Transformer-based language models and their training/adaptation lifecycle.

▶ Transformer pipeline

Text -> Tokenizer -> Embeddings + Positional Encoding
-> Stacked Self-Attention + FFN blocks -> Next-token probabilities

▶ Key concepts

  • Attention: weighted context aggregation between tokens.
  • Pretraining: broad language competence from large corpora.
  • Fine-tuning: domain/task adaptation for specificity and control.
  • Prompt engineering: behavior steering via instruction/context design.
  • Tokenization: subword representation balancing vocabulary and efficiency.

▶ Theory Deep Dive

LLMs model conditional probability over token sequences. Their effectiveness comes from scale (data/parameters/compute) and transformer attention, which captures long-range dependencies without recurrence. In practice, quality depends not just on base model size but also on retrieval context, prompt structure, decoding strategy, and alignment constraints.

  • Context window: bounded memory requiring careful context prioritization.
  • Alignment: safety and helpfulness tuning shapes behavior.
  • Grounding: external retrieval reduces hallucinations in factual tasks.

🌟 GPT-style models

Decoder-only transformers generate autoregressively by repeatedly sampling/choosing the next token conditioned on prior tokens.
