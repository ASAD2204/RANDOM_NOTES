MLOps (Machine Learning Operations)

Operational discipline that keeps models reliable after deployment.

▶ Core MLOps concerns

  • Model versioning and registry
  • Experiment tracking and reproducibility
  • Monitoring performance, latency, and cost
  • Data drift and concept drift detection
  • CI/CD for training and deployment workflows

▶ Theory Deep Dive

Model quality decays if environment assumptions change. MLOps treats this as a control problem: detect deviation, diagnose root cause, and trigger safe remediation (rollback, retraining, feature fixes). It aligns data engineering, model engineering, and platform operations into one lifecycle.

  • Reproducibility: same code + data + config should reproduce results.
  • Observability: collect model, feature, and business outcome metrics.
  • Governance: enforce lineage, approvals, and rollback policies.

▶ Typical tools

MLflow, Weights & Biases, DVC, Kubeflow, Airflow, GitHub Actions, Prometheus/Grafana, Evidently.
