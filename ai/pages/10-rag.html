Retrieval Augmented Generation (RAG)

Grounding language models with external knowledge retrieval for factual reliability.

▶ Architecture

User Query -> Embedding Model -> Vector DB -> Retriever
                    -> Prompt Constructor -> LLM -> Final Response

▶ Core components

  • Chunking strategy controls context granularity and recall quality.
  • Embeddings project text into semantic vector space.
  • Similarity search retrieves top-k likely relevant chunks.
  • Reranking improves precision before prompt construction.
  • Prompt construction determines how evidence influences generation.

▶ Theory Deep Dive

RAG separates knowledge storage from generation. The LLM provides language reasoning; retrieval provides current, domain-specific evidence. This modularity improves update speed and factual grounding without expensive full model retraining. The critical challenge is retrieval precision: irrelevant context can degrade output even with a powerful model.

  • Recall vs precision trade-off: too little context misses facts; too much context adds noise.
  • Chunking semantics: boundaries should preserve meaning units.
  • Evaluation: measure retrieval quality and answer faithfulness separately.

▶ Tooling ecosystem

LangChain, LlamaIndex, FAISS, Pinecone, and ChromaDB are common choices for orchestration and vector retrieval layers.
