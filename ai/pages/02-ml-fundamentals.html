Machine Learning Fundamentals

Core learning paradigms, pipeline logic, and theoretical principles behind model performance.

▶ Learning paradigms

  • Supervised learning: learn mapping from labeled examples for classification/regression.
  • Unsupervised learning: discover latent structure in unlabeled data.
  • Reinforcement learning: optimize sequential decisions using rewards.

▶ Typical ML pipeline

Data Collection -> Data Preprocessing -> Model Training
-> Validation/Evaluation -> Deployment -> Monitoring -> Iteration

The pipeline is cyclic. Production feedback should continuously refine features, labels, and training strategy.

▶ Theory Deep Dive

ML can be viewed as statistical risk minimization. During training, we minimize empirical loss on samples, hoping it approximates true future loss. This approximation depends on sample quality, feature relevance, and distribution stability. Overfitting appears when the model captures noise rather than signal.

  • Generalization: performance on unseen data is the real objective.
  • Bias-variance: simpler models may underfit, complex models may overfit.
  • Leakage control: preprocessing must be fit only on training partitions.
