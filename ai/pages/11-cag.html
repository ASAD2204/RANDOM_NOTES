Context Augmented Generation (CAG)

Using memory and contextual signals to produce coherent multi-turn responses.

▶ CAG core ideas

  • Context building from dialogue state, user profile, and task state.
  • Memory layers: short-term conversation memory and long-term memory.
  • Window management: summarize/rank/trim context for limited token budgets.
  • Knowledge injection: enforce domain rules and constraints inside prompts.

▶ CAG vs RAG

RAG: retrieval from external knowledge base is central
CAG: context-state construction and memory continuity are central
Practical systems often combine both.

▶ Theory Deep Dive

CAG treats conversation as a stateful process rather than isolated queries. The system decides what historical context to preserve, summarize, or discard to maximize relevance. This is crucial in workflows where user intent evolves over multiple turns. High-quality CAG requires careful memory policies to avoid stale or contradictory context.

  • Recency vs importance: recent messages are not always most relevant.
  • Compression: summarization preserves key facts within token limits.
  • Consistency: memory updates should maintain factual and persona stability.
